# 인플루언서 에이전트

크리에이터 한 명의 정보를 입력하면 전략 분석 → 콘텐츠 기획 → 4주 실행계획까지 생성. 성과를 입력하면 전략 재수립.
회사 조직(회장 → 경영 → 부서 → 직무)을 LLM 멀티 에이전트로 모델링, 프레임워크 없이 맨 API로 구현.

> **AI 협업 방식**
>
> 코드와 문서(ADR 포함)는 AI(Claude)가 작성. 사람의 몫은 셋.
> - 목표·방향 설정
> - AI가 잘못된 방향으로 가거나 오류를 낼 때의 교정
> - 세션마다 리뷰·ADR을 남겨, 기억 없이 시작하는 AI가 이전 맥락을 잇게 한 연속성 장치
>
> AI 생성 코드도 테스트·출력 검증·실 LLM e2e 통과분만 반영.

- 언어: Python 3.11+ · 프레임워크 없음
- 모델: 생성 gpt-4o · 품질 채점 Groq · 검색 Serper
- 인터페이스: CLI + 웹 (FastAPI + SSE)
- 테스트: 392 passing (391 + 라이브 1건은 API 키 없으면 skip)

### 미리보기

**관리 대시보드** · 이번 주 할 일 / 4주 로드맵 / 산출물 / 매니저 알림 (실시간 SSE)

![관리 대시보드](docs/assets/dashboard.png)

![대화형 인터뷰](docs/assets/chat.png)

**결과 산출물** · 컨셉 3종 / 영상 아이디어 5개 / 4주 캘린더 / 해시태그 (산출물 5종)

![결과 산출물](docs/assets/output.png)

## 1. 개요

1인 크리에이터는 콘텐츠는 만들지만 전략을 짜 주는 기획사가 없음. 단일 프롬프트로 "전략 짜줘"를 호출하는 대신, **기획사의 조직 구조를 모델로 차용.**

```mermaid
flowchart TD
    A["사람 (회장)<br/>목표·방향 결정 · 촬영/업로드 · 최종 승인"]
    B["CEO (경영)<br/>목표 해석 · 부서 임명 · 결과 취합 · 품질 검토"]
    C["기획 본부<br/>워커 오케스트레이션 · 결과 정제"]
    A -->|목표만 하달| B
    B -->|목적+목표 할당| C
    C -->|작업 지시| D["대상분석"]
    C --> E["경쟁분석"]
    C --> F["플랫폼추천"]
    C --> G["컨셉기획"]
```

상위 레이어는 "무엇을/왜", 하위 레이어는 "어떻게" 담당. 각 산출물은 검증 레이어(규칙 검사 + LLM 품질 채점) 통과 후 다음 단계로 전달.

> **범위 (정직 표기)**
>
> 현재 구현은 4계층 중 **수직 슬라이스 1개**(기획 본부). 마케팅·재무·법무 본부 등 가로 확장은 설계/로드맵 단계.
> 기본 동작은 고정 흐름의 **잘 구조화된 워크플로우**. 여기에 모델이 도구(워커)를 스스로 호출하는 **옵트인 자율 루프**(CLI `--autonomous` / 웹 토글) 병존. (정체성 논의: ADR 029)

## 2. 현재 상태 — 1장(MVP) 완료, 실제 작동

전부 실행 가능한 기능.

| 기능 | 설명 |
|------|------|
| 대화형 인터뷰 | CEO가 질문하며 대상자 정보 수집(이름/직업/특기/성격/타겟/SNS/목표). 폼 입력(`--legacy`) 지원 |
| 4단계 분석 | 대상분석(강점/약점/차별점) → 경쟁분석(포지셔닝 + 실제 구독자 수, 웹검색) → 플랫폼추천(1·2순위 + 근거) → 컨셉기획(컨셉 3 + 아이디어 5 + 4주 캘린더 + 촬영가이드 + 해시태그) |
| 품질 게이트 | 출력 검증(필수 섹션/이름 포함/금지표현/외래문자) + LLM 품질 채점(0~100), 미달 시 재시도 |
| 종합 리포트 | CEO가 4단계 결과를 6섹션 보고서로 종합(결론/핵심결정/강점기회/4주 로드맵/지금 할 일/KPI) |
| 실행계획 추출 | 캘린더 → 주차별 할 일, 리포트 → 다음 액션/KPI를 JSON으로 (`GET /api/plan/{이름}`) |
| 웹 대시보드 | 관리 중심 화면(이번 주 할 일 / 4주 로드맵 / 다음 액션) + 실시간 진행(SSE) |
| 재분석 루프 | 성과/피드백 입력 → CEO가 재실행 대상 판단 → 부분 재실행 |
| 매니저 알림 | 주간 실행 카드 / 진행 보고 / 성과 입력 요청 / 완료 요약 |

**검증 수치 (레포에서 확인 가능)**
테스트 392 passing (391 + 라이브 1건 skip) · 워커 4종 실 LLM E2E 품질 85~90/100 (대상90/경쟁85/플랫폼90/컨셉85, gpt-4o 생성 + LLM 저지 채점) · 의사결정 기록(ADR) 35건 (매 결정의 근거와 세션 간 연속성 확보용).

**최근 진행**
도구 호출 루프 프리미티브(`call_llm_tools` + `ToolExecutor` + `agent_loop`) 구현으로 모델이 분석 순서를 스스로 정하는 자율 루프(`CEO.run_autonomous`) 추가(선형과 병존) 및 CLI·웹 연결. 가상 인물 '사용자 여정' 평가(10단계 채점, 16/20 PASS)로 산출물 실사용 실행가능성 점검·보정.

## 3. 사용법

### 설치
```bash
pip install requests python-dotenv fastapi uvicorn
```
`.env`에 API 키:
```
OPENAI_API_KEY=...
GROQ_API_KEY=...
SERPER_API_KEY=...
```

### CLI (src/ 기준)
```bash
cd src

python main.py                              # 대화형 인터뷰 (기본)
python main.py --legacy                     # 7필드 폼 입력
python main.py --dry-run                    # API 없이 프롬프트 파일만 출력
python main.py --reanalyze --name 김민수      # 성과/피드백 기반 재분석
python main.py --init-performance --name 김민수   # 성과기록 템플릿 생성
python main.py --init-feedback --name 김민수      # 피드백 템플릿 생성
```

### 웹
```bash
cd src
python -m uvicorn api.main:app --port 8000
# http://127.0.0.1:8000
```

### 테스트
```bash
cd src
python -m pytest -q          # 392 passing (391 + 라이브 1건은 API 키 없으면 skip)
```

### 결과물 위치
```
src/outputs/{이름}/
├── 산출물/
│   ├── 01_대상분석.md  02_경쟁분석.md  03_플랫폼추천.md  04_컨셉기획.md
│   └── 최종리포트.md          (6섹션 종합)
├── 인수인계/
├── 성과기록.md / 피드백.md     (재분석 입력용, 1차 실행 후 자동 생성)
└── .system/                   (CEO 상태/계획, 에이전트 작업, 보고 이력, 로그)
```

## 5. 로드맵 — 1장 완료 · 2장 진행 중

- **1장 (완료)** · 전략 생성 워크플로우 (인터뷰 → 4단계 분석 → 종합 리포트 → 재분석 루프)
- **2장 (진행 중)** · 하네스에 자율 루프 확장 (도구 호출 + 루프 + 작업기억 + 가드레일). 실제 미용사 1명 적용도 병행 진행 중.
- **3장 (이후)** · 다부서 가로 확장 (마케팅·재무·법무 본부, 부서 간 협업 / Cross-review)

2장 세부 항목:

| 우선순위 | 항목 | 내용 |
|:---:|------|------|
| 1 | 측정·실험 설계 | KPI + 콘텐츠 변수 태깅(주제/빈도/시간대/형식) → 성과 귀속 + 의사결정 로그 |
| 2 | 컨시어지 적용 | 운영자(개발자)가 시스템을 돌리고 미용사는 결과 수령, 4주 운영 |
| 3 | 최소 자율 루프 | 성과 데이터를 보고 다음 행동을 스스로 결정하는 도구 호출 루프 (1장을 도구로 감쌈) |
| 4 | 판단 검증 (트랙0) | CEO의 LLM 판단이 조용히 폴백하지 않는지 계측 |

**성공 기준 (2축)**
(실사용) 실제 팔로워 성장 · (검증) 측정 가능한 실험 설계 + 의사결정 로그. "데이터가 쌓이면 검증된다"가 아니라 "통제된 실험이 검증을 만든다"는 원칙으로 cross-AI 리뷰(Claude+GPT)를 거쳐 확정.
