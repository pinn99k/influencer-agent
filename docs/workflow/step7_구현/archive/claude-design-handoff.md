# Claude Design 핸드오프 — 인플루언서 에이전트 웹 UI
date: 2026-05-14
spiral: 3 (CLI 검증 완료 후 적용)
status: 설계 준비 (Spiral 0-B, 1, 2 완료 후 구현)

> 이 문서는 Claude Design(또는 디자이너)에게 웹 UI 구현을 의뢰하기 위한 핸드오프 문서다.
> 기능·데이터·흐름은 여기에 고정. 시각 디자인 판단은 Claude Design이 한다.

---

## 1. 이 프로그램이 무엇인가

**AI 기획사다.**

핑구(회장)가 목표와 대상자 정보를 설정하면, CEO 에이전트가 그 인플루언서를 자율적으로 경영한다.
분석은 시작일 뿐이다. 이후 CEO는 스스로 판단해서 다음 액션을 결정하고, 하위 에이전트를 임명하고, 결과를 평가하고, 콘텐츠 전략을 업데이트한다.

핑구는 목표를 설정하고 중요한 결재만 한다. 나머지는 CEO가 알아서 한다.

**CEO는 성과로 평가된다. 못하면 교체될 수 있다.**

```
핑구 (회장)
  — 목표 설정 + 대상자 정보 입력 → 이후 개입 최소화
  — 보고 조건 10개 해당 시에만 결재
  — 촬영·편집·업로드 등 실행 담당
  — CEO 성과 평가 → 기준 미달 시 교체

CEO 에이전트 (자율 운영)
  — 목표를 항상 인지한 채 스스로 계획 수립·실행
  — 하위 에이전트 임명 → 결과 평가 → 피드백 → 승인
  — 성과 데이터 수집 → 전략 수정 → 루프
  — 성과 기록됨, 책임 있음

하위 에이전트 (CEO 지시에만 응답)
  현재: 대상분석 / 경쟁분석 / 플랫폼추천 / 컨셉기획
  추후: 스크립트 / 성과분석 / 콘텐츠평가 / ...
```

**CEO 자율 루프:**
```
목표 확인
→ 현재 상태 파악 (구독자 수, 최근 콘텐츠 성과)
→ 다음 액션 결정 (스스로 판단)
→ 해당 에이전트 실행 → 결과 평가 → 피드백
→ 핑구에게 실행물 전달 (필요 시)
→ 성과 수집 → 루프
```

---

## 2. UI의 역할

이 UI는 **에이전트 실행 버튼이 아니다.**

CEO가 지금 무엇을 하고 있는지 핑구가 볼 수 있는 창이고,
핑구가 결재해야 할 것이 생겼을 때 받는 알림 창이다.

CEO는 핑구가 브라우저를 보고 있든 없든 계속 돌아간다.

## 3. 화면 구조

### 전체 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  상단 바: 관리 중인 인플루언서 목록 + CEO 전체 상태       │
├──────────────────────┬──────────────────────────────────┤
│                      │                                  │
│  왼쪽: 실시간 활동   │  오른쪽: 대시보드                │
│                      │                                  │
│  지금 어떤 에이전트  │  CEO 현재 계획                   │
│  가 뭘 하고 있는지   │  최근 완료된 작업                │
│  실시간으로 표시     │  성과 지표                       │
│                      │  핑구 결재 대기 항목             │
│                      │                                  │
└──────────────────────┴──────────────────────────────────┘
```

### 진입 화면 (최초 1회만)

CEO에게 목표와 대상자 정보를 줄 때만 등장하는 화면.
이후에는 대시보드가 메인이다.

**입력 항목:**
- 대상자 7개 필드 (이름 / 직업 / 특기 / 성격 / 타겟 연령대 / SNS 경험 / 목표)
- CEO에게 줄 최종 목표 ("6개월 내 유튜브 구독자 1만명")

**버튼:** "CEO 임명" (경영 시작)
→ 이후 이 화면은 다시 나오지 않음. 목표/대상자 수정은 대시보드에서.

## 4. 핵심 컴포넌트

### C-01: ActivityFeed (실시간 활동 패널)

```
역할:    CEO와 하위 에이전트가 지금 무엇을 하고 있는지 실시간 표시
위치:    화면 왼쪽 고정
데이터:  Session Manager → SSE 스트리밍
표시 예:
  [14:32] CEO — 다음 액션 판단 중
  [14:32] 경쟁분석 에이전트 — LLM 호출 중
  [14:33] 경쟁분석 에이전트 — 검증 중
  [14:33] CEO — 결과 평가 중
  [14:34] CEO — 핑구 보고 대기 (조건 5번)
```

핑구가 브라우저를 열었을 때 CEO가 이미 무언가를 하고 있다면 바로 보인다.

### C-02: CEODashboard (CEO 상태 대시보드)

```
역할:    CEO의 현재 계획 + 최근 결정 + 성과 지표 표시
표시:
  현재 목표:      "6개월 내 구독자 1만명 (D-142)"
  CEO 현재 상태:  "경쟁분석 재실행 중 (3개월 경과)"
  최근 완료:      컨셉기획 업데이트 (2026-05-10)
  성과:           구독자 230명 (+30, 이번 주)
  다음 예정:      스크립트 초안 작성 (내일)
```

### C-03: PendingApprovals (핑구 결재 대기)

```
역할:    회장 보고 조건 해당 항목을 명확하게 표시
표시:    결재 대기 카드 목록 (조건 번호 + 요약 + 선택지)
인터랙션: 카드 클릭 → BriefingModal 열림
알림:    결재 대기 항목이 생기면 배지로 표시
```

이게 없으면 CEO가 멈춰있다. 핑구가 가장 먼저 확인해야 하는 영역.

### C-04: BriefingModal (회장 보고 모달)

```
역할:    회장 보고 발생 시 A/B 선택 요청
블로킹:  전체 화면 오버레이 (닫기 없음 — 반드시 선택)
내용:    보고 조건 / 현황 요약 / 선택지 A·B (장단점) / CEO 권고안
버튼:    "A 선택" / "B 선택"
→ 선택 후 POST /decision/{job_id} → CEO 루프 재개
```

### C-05: AgentActivityBar (에이전트 진행 상태)

```
역할:    현재 실행 중인 작업의 에이전트별 상태 표시
상태:    WAITING / RUNNING / DONE / FAILED
표시:    ActivityFeed 상단 또는 대시보드 내 위젯
```

### C-06: ReportViewer (산출물 뷰어)

```
역할:    완료된 에이전트 결과물 열람
접근:    대시보드에서 "최근 완료" 항목 클릭 시
탭:      대상분석 / 경쟁분석 / 플랫폼추천 / 컨셉기획 / 최종리포트
액션:    다운로드 (.md)
```

### C-07: GoalEditor (목표/대상자 수정)

```
역할:    핑구가 CEO에게 주는 목표 또는 대상자 정보 수정
접근:    대시보드 설정 영역
주의:    수정 시 CEO에게 변경 사항 전달됨 (회장 보고 조건 3번 가능)
```

## 5. Session Manager — UI만이 아닌 백엔드 컴포넌트

> UI 변경만으로는 구현 불가. `src/api/session_manager.py` 신규 구현 필요.

### 역할

CEO가 핑구의 브라우저 상태와 무관하게 돌아가려면 Session Manager가 필요하다.

```
Session Manager가 관리하는 것:
  - CEO 현재 상태 (계획 중 / 실행 중 / 대기 중 / 완료)
  - 실행 중인 job 목록 (job_id → 에이전트명 + 상태 + 시작시각)
  - 핑구 결재 대기 항목
  - 이벤트 큐 → SSE로 UI에 스트리밍
```

### API 엔드포인트

| 메서드 | 경로 | 역할 |
|--------|------|------|
| `POST` | `/start` | CEO 임명 (최초 1회) |
| `GET` | `/status` | CEO + 실행 중인 job 전체 상태 |
| `GET` | `/stream` | SSE — 전체 이벤트 실시간 스트림 |
| `GET` | `/stream/{job_id}` | SSE — 특정 job 이벤트만 |
| `DELETE` | `/job/{job_id}` | 실행 중인 job 취소 |
| `POST` | `/decision/{job_id}` | 회장 보고 응답 |
| `PATCH` | `/goal` | 목표 수정 |

### SSE 이벤트 구조

```json
{"timestamp": "...", "type": "agent_start",    "agent": "경쟁분석", "job_id": "..."}
{"timestamp": "...", "type": "agent_done",     "agent": "경쟁분석", "job_id": "..."}
{"timestamp": "...", "type": "ceo_thinking",   "detail": "다음 액션 판단 중"}
{"timestamp": "...", "type": "briefing",       "condition": 5, "job_id": "..."}
{"timestamp": "...", "type": "job_failed",     "agent": "경쟁분석", "retry": 1}
```


## 9. 유지보수 고려

- **에이전트 추가:** ActivityFeed(C-01)와 SSE 이벤트 타입만 추가. 나머지 구조 변경 없음.
- **성과 지표 추가:** CEODashboard(C-02) 위젯 확장. Session Manager에 지표 필드 추가.
- **인플루언서 다수 관리:** Session Manager가 인플루언서별 CEO 인스턴스를 관리하는 구조로 확장.
- **인증:** MVP는 로컬 전용. 외부 공개 시 API Key 헤더 추가.
- **마크다운 렌더링:** marked.js + DOMPurify (XSS 방지) 필수.
