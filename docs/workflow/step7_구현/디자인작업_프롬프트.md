먼저 docs/workflow/step7_구현/핸드오프_v2.md 를 읽으세요. 거기에 제약·계약·
인터페이스가 전부 있습니다.

## 작업
src/api/static/views.js 와 style.css 를 재설계하세요.
프론트는 MVC로 분리되어 있고, 당신은 View(views.js) + 스타일(style.css)만
수정합니다. api.js / model.js / actions.js 는 절대 건드리지 마세요.

## 반드시 지킬 것 (핸드오프 C-1~C-5)
1. 화면 height:100vh 고정, 내부 패널만 스크롤 (페이지 전체 스크롤 금지)
2. CEO 채팅을 대시보드에서도 상시 노출 (분석 완료 후에도 대화 가능)
3. 운영/재분석/새대상자 양방향 네비게이션
4. View 순수성: State에서 읽기만, 동작은 Actions.xxx() 호출만.
   fetch/EventSource/localStorage 직접 호출 금지. 새 동작 필요하면 주석으로
   "// NEEDS-ACTION: ..." 표시 (개발자가 actions.js에 구현).
5. 한국어 UI. 코드/주석에 한자 절대 금지 — 한글과 영문, 숫자만 사용.

## 사용 가능한 것
- 데이터: State.* (핸드오프 2-1)
- 동작: Actions.* (핸드오프 2-2)
- 헬퍼: h(), renderMd(), avatar(), pill() (config.js, 핸드오프 2-4)
- 진입점 render()는 main.js에 있음 — State.appointed로 분기하는 구조 유지

## 산출

- style.css
