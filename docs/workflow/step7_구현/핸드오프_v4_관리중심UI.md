## 1. 새로 생기는 데이터 (코덱스가 채움 — 당신은 읽기만)

### State.plan (신규)
```
State.plan = {
  weeks: [ { num: 1, items: ["릴스: 가을 헤어컬러 변신 30초", "..."] }, ...4개 ],
  next_actions: ["오늘: 프로필/바이오 세팅", "...", "..."],   // 0~3개
  kpi: ["팔로워: 300+", "콘텐츠: 12개", "핵심 KPI: 조회 1만+"] // 0~N개
}
```
- 분석 완료 후 자동 로드됨 (Actions.loadPlan 이 채움). 미완료/실패 시 null.
- weeks 의 각 item 은 캘린더에서 뽑은 한 줄 텍스트 (촬영할 콘텐츠).

### 최종 리포트 (의미 변경)
- reports[] 중 "최종리포트.md" 가 이제 CEO LLM 종합 보고서다 (단순 합본 아님).
  6개 섹션: 한 줄 결론 / 핵심 결정 3가지 / 강점과 기회 / 4주 실행 로드맵 /
  지금 당장 할 3가지 / 성공 지표(30일).
- 이걸 "전략 브리핑"으로 격상 — 산출물 목록의 1/5가 아니라 별도 진입점으로.

## 2. 요구 화면 구조 (관리 중심)

우선순위: 사용자가 화면에 들어오면 "지금 뭘 해야 하는지"가 가장 먼저 보여야 함.

### 완료 상태(ceoState=completed) 대시보드 중앙 — 새 구성
```
[ 이번 주 할 일 ]   <- State.plan.weeks 중 현재 주(기본 Week 1) 체크리스트
   - [ ] 릴스: 가을 헤어컬러 변신 30초
   - [ ] ...
   (주차 전환 탭: Week 1 2 3 4)

[ 4주 로드맵 ]      <- weeks 4개를 가로 타임라인/스텝퍼로. 각 주 항목 수 표시
   Week1 ●━ Week2 ○━ Week3 ○━ Week4 ○      진행률 막대

[ 다음 액션 ]       <- State.plan.next_actions 강조 카드 (지금 당장 할 것)

[ 전략 브리핑 열기 ] <- 최종리포트.md 를 모달/뷰로 (기존 viewReport 재사용 가능)
```

### 분석 4개(대상/경쟁/플랫폼/컨셉) — 격하
- "근거 자료" 섹션으로 접어서 우측 또는 하단. 펼치면 기존 산출물 카드.
- 완전히 숨기지 말 것 (전략의 근거 — 신뢰도). 단 1순위 자리는 양보.

### 분석 중 상태(ceoState=running) — 유지
- 기존 작업 현황(에이전트 타일 + 타임라인) 그대로. plan 은 아직 null 이므로
  "분석이 끝나면 이번 주 할 일이 여기 표시됩니다" 플레이스홀더.

### KPI
- 우측 레일 상단 또는 목표 카드에 State.plan.kpi 를 작은 지표로.
