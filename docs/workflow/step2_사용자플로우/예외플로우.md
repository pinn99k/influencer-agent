# Step 2 — 예외플로우
updated: 2026-05-14 (세션 13)

## 예외 1: 정보 부족
하위 에이전트가 대상자 정보 부족 감지 시

```
하위 에이전트 → "정보 부족" 플래그 + 필요한 정보 명시
→ 경영 에이전트 수신
→ 핑구에게 추가 질문 (회장 보고 형식)
→ 핑구 답변
→ 컨텍스트 업데이트 후 해당 에이전트 재호출
```

보고 형식:
```
## 추가 정보 요청
에이전트: {에이전트 이름}
필요한 정보: {무엇이 필요한가}
이유: {왜 필요한가}
질문: {핑구에게 묻는 내용}
```

## 예외 2: 에이전트 출력 검증 실패
출력이 FORMAT 기준 미충족 시

```
경영 에이전트: 출력 검증 실패 감지
→ 동일 에이전트 1회 재호출 (에러 피드백 포함)
→ 재시도 성공 → 정상 플로우 복귀
→ 재시도 실패 → 회장 보고 (보고 조건 5번)
```

## 예외 3: 회장 보고 조건 감지
경영 에이전트가 보고 조건 10개 중 하나 감지 시

```
경영 에이전트: 즉시 작업 중단
→ 핑구에게 브리핑
→ 핑구 결정 수신
→ A 또는 B 선택에 따라 재개 또는 종료
```

보고 형식:
```
## 회장 보고
보고 조건: {조건 번호 + 내용}
현재 상황: {무슨 일이 발생했는가}
선택지:
  A. {선택지 A}
  B. {선택지 B}
결정 요청: A 또는 B 선택 후 진행
```

## 예외 4: 경영 에이전트 목표 해석 실패
목표·목적이 불명확하여 실행 계획 수립 불가 시

```
경영 에이전트 → 핑구에게 목표 재확인 요청
→ 핑구 답변
→ 목표 재해석 후 실행 계획 수립
```

## 예외 5: CEO 성과 기준 미달 (누적 실패)
에이전트 반복 실패, 목표 지속 미달 등 CEO가 목표를 달성하지 못하는 상황

```
회장 보고 (조건 5 또는 조건 10) 누적 발생
→ 핑구가 CEO 성과 평가
→ 판단: 계속 또는 교체
   - 계속: 목표 재설정 후 루프 재개
   - 교체: CEO 프롬프트 리셋 또는 모델 교체 (핑구 승인 필수)
```

> CEO 교체는 반드시 핑구가 결정한다. 경영 에이전트 스스로 자기 교체를 결정하는 것은 금지.

## 공통 원칙
- 모든 예외 상황은 경영 에이전트가 핑구에게 보고 (하위 에이전트 직접 보고 금지)
- 하위 에이전트는 판단 불가 상황 시 "플래그만" 올린다
- 예외 처리 결과는 context에 기록
- max_retry = 1 (에이전트당 최대 재시도 1회)
