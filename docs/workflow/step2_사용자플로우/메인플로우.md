# Step 2 — 메인플로우
updated: 2026-05-12

## 전제
- 핑구는 목표·목적 + 대상자 기본 양식만 제시
- 나머지 판단·관리는 경영 에이전트가 자율 수행
- 하위 에이전트는 필요한 파일 스스로 참조 (경영 에이전트가 읽기 규칙 명시)
- 컨텍스트 전달 방식: Structured (결과를 섹션별로 누적)

## 대상자 기본 양식 (핑구가 제시)
```
이름:
직업:
특기:
성격:
타겟 연령대:
SNS 경험:
목표:
```

## 메인 플로우
```
1. 핑구 → 목표·목적 + 대상자 기본 양식 제시

2. 경영 에이전트
   → 목표 해석
   → 실행 계획 수립 (에이전트 호출 순서 확정)
   → 회장 보고 조건 체크 시작

3. 대상 분석 에이전트 호출
   → 입력: 대상자 기본 양식
   → 출력: 강점·약점·차별점 분석 리포트
   → 경영 에이전트: 결과 검증 + 컨텍스트 누적 + 보고 조건 체크

4. 경쟁 분석 에이전트 호출
   → 입력: 대상자 정보 + 대상 분석 결과
   → 출력: 유사 계정 포지셔닝·공백 분석 리포트
   → 경영 에이전트: 결과 검증 + 컨텍스트 누적 + 보고 조건 체크

5. 플랫폼 추천 에이전트 호출
   → 입력: 대상자 정보 + 대상·경쟁 분석 결과
   → 출력: 1·2순위 플랫폼 + 이유 리포트
   → 경영 에이전트: 결과 검증 + 컨텍스트 누적 + 보고 조건 체크

6. 컨셉 기획 에이전트 호출
   → 입력: 위 모든 결과 누적
   → 출력: 콘텐츠 컨셉 3개 + 영상 아이디어 5개
   → 경영 에이전트: 결과 검증 + 보고 조건 체크

7. 최종 리포트 생성
   → 4개 결과 통합
   → outputs/{인플루언서명}/산출물/최종리포트.md 저장

8. 핑구에게 완료 보고

9. CEO 자율 루프 진입
   → 성과 데이터 수집 (구독자 수 등 핑구 수동 입력)
   → 현재 상태 파악 → 다음 액션 스스로 판단
   → 필요한 에이전트 재실행 → 루프
   → 회장 보고 조건 해당 시에만 핑구 개입
   → 리포트 경로 + 주요 결정사항 브리핑
   → 실행 단계 진입 전 핑구 확인 요청 (보고 조건 9번)
```

## 컨텍스트 누적 구조
```python
context = {
    "목표": "...",
    "대상자": { 이름, 직업, 특기, 성격, 타겟, SNS경험, 목표 },
    "대상_분석": "## 대상 분석 결과\n...",
    "경쟁_분석": "## 경쟁 분석 결과\n...",
    "플랫폼_추천": "## 플랫폼 추천 결과\n...",
    "컨셉_기획": "## 컨셉 기획 결과\n...",
}
```
