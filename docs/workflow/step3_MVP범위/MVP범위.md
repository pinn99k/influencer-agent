# Step 3 — MVP 범위
updated: 2026-05-12 (세션 3 수정)

> **현황(2026-06-15):** MVP 범위(아웃풋 4개 + 품질 기준)는 1장에서 전부 구현·검증 완료. 실시간 진행은 docs/workflow/status.md(세션 36) 참조.
>
> 이 문서는 구현 범위의 기준선이다. 흔들릴 때 여기 보고 판단한다.
> MVP ≠ 최종 목표. MVP는 검증 가능한 최소 단위.

---

## 인풋

| 필드 | 설명 |
|------|------|
| 이름 | 크리에이터 이름 (가명 가능) |
| 직업 | 현재 직업 또는 전문 분야 |
| 특기 | 잘하는 것 / 두드러지는 능력 |
| 성격 | 외향/내향, 말하기 스타일 등 |
| 타겟 연령대 | 주로 어필하고 싶은 연령대 |
| SNS 경험 | 현재 운영 중인 채널/계정 현황 |
| 목표 | 6개월~1년 내 달성하고 싶은 것 |

---

## 아웃풋 4개 (이것만)

| # | 에이전트 | 아웃풋 |
|---|---------|--------|
| 1 | 대상 분석 | 강점 / 약점 / 차별점 |
| 2 | 경쟁 분석 | 경쟁 계정 포지셔닝 |
| 3 | 플랫폼 추천 | 인스타 / 유튜브 / 틱톡 + 이유 |
| 4 | 컨셉 기획 | 콘텐츠 컨셉 3개 + 영상 아이디어 5개 |

---

## 에이전트 구조

```
👤 핑구 (회장)
    ↓ 대상자 기본정보 + 목표 입력
🤖 경영 에이전트 (CEO)
    ↓ 순서대로 직접 임명
🤖 대상 분석 → 🤖 경쟁 분석 → 🤖 플랫폼 추천 → 🤖 컨셉 기획
    ↓
경영 에이전트: 결과 취합 → 회장 보고 조건 체크 → 최종 리포트 출력
```

- 컨텍스트 전달: Structured (이전 에이전트 결과 누적해서 다음 에이전트 인풋으로)
- 저장: `outputs/report_{name}_{date}.md`

---

## 품질 기준

> 구조 검증 + 실사용 품질 **동시 충족**. 둘 중 하나만이면 MVP 실패.

- **구조 검증:** 에이전트 4개가 순서대로 실행되고, 컨텍스트가 누적 전달되며, CEO가 취합·보고 조건을 체크한다
- **실사용 품질:** 핑구가 직접 읽고 "이걸 크리에이터에게 줄 수 있다"고 판단되는 수준

### 모델 전략

| 단계 | 모델 | 기준 |
|------|------|------|
| MVP 구조 검증 | Groq llama3-70b (무료) | 기본 |
| 품질 NG 에이전트 | gpt-4o 또는 Claude API | 핑구 직접 판단 → NG 에이전트만 교체 |

> 전체 교체 아님. 병목 에이전트만 개별 교체.

## 회장 보고 조건 (MVP에서도 전면 적용)

경영 에이전트는 아래 조건 해당 시 작업 즉시 멈추고 회장 브리핑. 단독 결정 금지.

1. 비용 발생하는 모든 결정
2. 법적 리스크 감지
3. 초기 목표 변경 필요 시
4. 크리에이터 교체·컨셉 전면 수정 등 사람에게 영향
5. 에이전트 2회 재작업 후도 실패
6. 에이전트 간 결과 충돌
7. 실제 외부 사람 컨택 필요 시
8. 개인정보·결과 외부 공개 필요 시
9. AI 산출물이 실제 실행으로 넘어가는 시점
10. 회장 기대치와 결과가 크게 다를 것 같을 때

## 개발 로드맵 (나선형)

> 설계는 폭포수(Step 0→9 순서 고정). 구현은 나선형(기능 단위 개발→검증→다음).

| Spiral | 내용 | 완료 조건 |
|--------|------|-----------|
| 0-A | CEO 에이전트 뼈대 + 프롬프트 파일 출력 | 핑구가 프롬프트 직접 테스트 후 품질 확인 |
| 0-B | CEO + Groq API 실제 연결 + 마크다운 저장 | end-to-end 실행 1회 성공 |
| 1 | 직무 에이전트 4개 실제 구현 (하나씩 교체) | 아웃풋 4개 실사용 품질 통과 |
| 2 | 품질 레이어 (검증기 + max_retry + 회장 보고) | 실제 인플루언서 1명 적용·기록 완료 |
| 3 | 웹 UI (Claude 디자인, UX 심플 최우선) | 입력→실행→결과 조회 웹에서 가능 |
