## 핵심 전환

| 항목 | V1 (현재) | V2 (확장) |
|------|----------|----------|
| 입력 | 7개 필드 폼 | CEO 대화형 인터뷰 (10턴+) |
| 에이전트 동작 | 프롬프트(지시서) 기반 결과물 생성 | 역할 가이드 + 자율 판단 + 질문/첨언 |
| 소통 방향 | 단방향 (입력 -> 출력) | 양방향 (에이전트 <-> CEO <-> 사용자) |
| 상태 인식 | 무상태 (매번 context dict 전달) | 파일 기반 상태 관리 (outputs/ 직접 읽기) |
| 사용자 인터페이스 | CLI 폼 입력 | CLI 인터뷰 -> 웹 채팅 (Phase 3) |
| 보고 | 최종 리포트 1회 | 매니저 에이전트 상시 알림/보고 |

## 아키텍처 요약

```
사용자 <-> InterviewEngine (10턴+ 대화)
           |
           v
         CEO (목표해석 + 에이전트 지휘 + 판단)
           |                              |
           v                              v
  PlanningDepartment             ManagerAgent (알림/보고)
    |    |    |    |
    v    v    v    v
  대상  경쟁  플랫폼  컨셉     (각 에이전트: AgentOutput 반환)
  분석  분석  추천    기획     (AgentContext: 접근범위 제한)
```

---

## 에이전트 소통 구조

```
하위 에이전트 -> CEO: AgentOutput (결과 + 질문 + 첨언)
CEO -> 하위 에이전트: AgentContext (접근범위 내 데이터 + CEO 요약)
CEO -> 사용자: 전략적 질문 에스컬레이션 + 최종리포트
CEO -> 매니저: 알림/보고 지시
매니저 -> 사용자: 주간카드, 진행보고, 성과입력 요청
```

## 개발 Spiral (나선형)

| Spiral | 내용 | 완료 기준 |
|--------|------|----------|
| 5-A | AgentOutput + AgentContext 기반 구조 | 4개 에이전트 AgentOutput 반환 + 접근권한 테스트 통과 |
| 5-B | 프롬프트 역할 가이드 + CEO 질문처리 | 프롬프트 전환 + Validator 테스트 통과 + 산출물 품질 유지 |
| 5-C | 대화형 인터뷰 엔진 | CLI 10턴 대화 -> subject dict 추출 성공 |
| 5-D | 매니저 + 루프 완성 | 주간카드 생성 + --reanalyze 실행 성공 |
| 5-E | 웹 통합 (시간 여유 시) | 웹 채팅 + 대시보드 동작 |

## 성공 기준

V2 확장 성공 = 아래 모두 충족:
1. 핑구가 CLI 인터뷰로 자연스럽게 정보 입력 가능
2. 에이전트가 질문/첨언을 포함한 결과물 반환
3. CEO가 질문을 자체 판단하거나 사용자에게 에스컬레이션
4. 매니저가 주간 할 일 카드 + 진행보고 생성
5. 재분석 모드로 전략 수정 가능
6. 기존 산출물 품질 유지 (E2E 80+)
