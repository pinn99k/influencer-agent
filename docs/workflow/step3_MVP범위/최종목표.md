## 왜 만드는가

인플루언서 1명을 관리하는 AI 기획사.
전략·분석·기획·문서화는 에이전트가, 실행(촬영·편집·업로드)은 사람이 한다.
기업도 경영팀이 콘텐츠를 직접 만들지 않는다. 그 구조를 AI로 구현한다.

---

## 핵심 철학

### AI vs 사람 역할 분리

```
AI 에이전트 영역:
  분석, 기획, 전략, 초안, 브리핑, 조율, 판단, 문서화

사람(핑구·크리에이터) 영역:
  촬영, 편집, 업로드
  외부 사람과 직접 소통 (브랜드, 팬, 협찬사)
  계약서 서명
  최종 방향 결정 (회장)
  실제 SNS 운영
```

AI가 실행 단계로 넘어가는 순간 반드시 회장(핑구) 승인을 받는다.

### 브리핑 정제 원칙

날 것의 결과가 위 레이어로 올라가는 일은 없다.
각 레이어는 자기 레벨의 언어로 정제해서 위로 올린다.

```
직무 에이전트  → "작업 결과" 정제      → 부서에 브리핑
부서 에이전트  → "전략적 시사점" 정제   → 경영에 브리핑
경영 에이전트  → "의사결정 포인트" 정제 → 회장에 브리핑 (조건 해당 시)
```

## 계층 구조 (4계층 — 완성 목표)

```
👤 핑구 (회장)
│  역할: 최종 방향 결정 · 지출·법적·실행 승인 · 브리핑 수신 후 판단
│  개입: 10개 보고 조건 해당 시만
│  실행: 촬영·편집·업로드·외부 소통·계약 서명
│
└── 🤖 경영 에이전트 (CEO)
    │  역할: 목표·목적 보유 / 부서에 목적+목표 할당 / 브리핑 취합·판단
    │  금지: 실무 직접 수행 / 회장 보고 조건 단독 결정
    │
    ├── 🤖 기획 본부        크리에이터 분석·전략 수립
    ├── 🤖 마케팅 본부      SNS 운영 전략·콘텐츠 실행 기획
    ├── 🤖 재무 본부        수익성 분석·예산·협찬 단가       (Phase 7)
    └── 🤖 법무·계약 본부   계약서 검토·법적 리스크 감지      (Phase 7)
```

> MVP에서는 경영 에이전트가 직무 에이전트 4개를 직접 임명 (부서 레이어 없음).
> Phase 4에서 부서 레이어가 경영과 직무 에이전트 사이에 추가된다.

## CEO 자율 경영 루프

CEO는 일회성 실행이 아니다. 목표 달성까지 스스로 루프를 돈다.

```
목표 확인
→ 현재 상태 파악 (구독자 수, 최근 콘텐츠 성과, 완료된 작업)
→ 다음 액션 스스로 판단
   예: "3개월 지남 → 경쟁 재분석 필요"
   예: "컨셉 반응 저조 → 컨셉 업데이트 필요"
→ 해당 에이전트 실행 → 결과 평가 → 피드백 or 승인
→ 핑구에게 실행 요청 (필요 시, 회장 보고 조건 9번)
→ 성과 데이터 수집
→ 루프
```

핑구가 브라우저를 보고 있든 없든 CEO는 계속 돌아간다.
핑구는 보고 조건 해당 시만 개입한다.

### CEO 책임과 교체 조건

CEO 성과는 기록되고 평가된다.

| 상황 | 조치 |
|------|------|
| 에이전트 2회 실패 | 회장 보고 (조건 5) → 핑구 판단 |
| 누적 목표 미달 | 핑구가 CEO 교체 결정 가능 |
| CEO 교체 | 프롬프트 리셋 또는 모델 교체 (핑구 승인) |

CEO는 이 사실을 시스템 프롬프트에서 인지한다 — "성과가 기록되고 평가된다."

## 브리핑 형식

### 직무 에이전트 → 부서 브리핑

```
[직무 브리핑]
에이전트: {직무명}
완료 작업: {무엇을 했는지 1줄}
핵심 결과: {3줄 이내 핵심만}
주요 발견: {예상과 달랐던 것·주목할 것}
한계·불확실성: {[추정] 포함 항목·신뢰도 낮은 부분}
부서 판단 필요 사항: {부서가 결정해야 할 것}
```

### 부서 에이전트 → 경영 브리핑

```
[부서 브리핑]
부서: {부서명}
완료 작업: {이번 할당 완료 요약}
전략적 시사점: {경영이 알아야 할 핵심 3가지}
타 부서 협업 내용: {있었다면 요약}
리스크: {감지된 리스크·불확실성}
경영 판단 필요 사항: {경영이 결정해야 할 것 — 없으면 "없음"}
회장 보고 필요 여부: {예/아니오 + 이유}
```

### 경영 에이전트 → 회장 브리핑 (조건 해당 시)

```
[회장 브리핑]
긴급도: {즉시/보통}
보고 사유: {10개 조건 중 해당 항목}
현황 요약: {지금까지 완료된 것 3줄}
판단 요청 사항: {회장이 결정해야 할 것}
선택지:
  A) {옵션A} — 장점: / 단점:
  B) {옵션B} — 장점: / 단점:
권고안: {경영 에이전트 추천 + 이유}
```

## 회장 보고 조건 10가지

경영 에이전트는 아래 조건 해당 시 즉시 멈추고 회장 브리핑. 단독 결정 절대 금지.

**지출·법적**
1. 비용 발생하는 모든 결정 (모델 전환, 유료 API, 툴 구독 등)
2. 법적 리스크 감지 (계약 조항 이상, 저작권 문제, 분쟁 가능성)

**전략·방향**
3. 초기 목표 변경 필요 시 (플랫폼, 타겟 연령대, 컨셉 전면 수정)
4. 크리에이터 교체·컨셉 전면 수정 등 사람에게 직접 영향

**교착·실패**
5. 에이전트 2회 재작업 후도 실패
6. 에이전트·부서 간 결과 충돌로 합의 불가

**외부·실행**
7. 실제 외부 사람 컨택 필요 시 (크리에이터·브랜드·협찬사)
8. 크리에이터 개인정보·분석 결과 외부 공개 필요 시
9. AI 산출물이 실제 실행(촬영·업로드·서명)으로 넘어가는 모든 시점

**기대치**
10. 경영 판단상 회장 기대치와 결과가 크게 다를 것 같을 때

## 재작업 조건

아웃풋에 아래 문제 발생 시 해당 에이전트에 재작업 지시 (최대 1회):

- 필수 섹션 누락
- 입력에 없는 정보를 [추정] 태그 없이 단정
- 직무 범위 초과 내용 포함
- 실제 계정명·인물명 언급

2회 재작업 후도 실패 → 경영 에이전트가 회장에게 보고 (조건 5번).

## 협업 프로토콜 (Phase 5 이후)

| 레이어 | 협업 가능 대상 |
|--------|--------------|
| 회장 | 경영 에이전트 |
| 경영 에이전트 | 회장 · 각 부서 |
| 부서 에이전트 | 경영 · 타 부서 |
| 직무 에이전트 | 소속 부서만 |

직무 에이전트는 부서 밖으로 직접 연락 불가. 반드시 부서를 통한다.

### 부서 간 협업 흐름 (Phase 5)

```
기획 본부가 재무 본부에 협업 요청 시:
1. 기획 본부 → 경영에 CC 포함 협업 요청 발송
2. 재무 본부 → 요청 수행 후 기획 본부에 결과 반환 (경영에 CC)
3. 기획 본부 → 결과 취합·정제 후 경영에 브리핑
경영은 흐름을 인지하되 직접 개입하지 않는다.
```
