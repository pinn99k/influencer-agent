## 4주 컨시어지 타임라인 (병행)

```
D-7~D-1 (준비): 측정 레이어(M) 구축 + 실제 미용사 정보 수집 + 1장로 온보딩 산출물 생성
Week 1: 미용사 SNS 세팅 + 첫 콘텐츠 3~4개(주간카드 전달). 핑구는 최소 자율 루프(T) 빌드
        (1주차 운영은 저터치 -> 코딩 시간 확보)
Week 2: 성과 입력 -> 최소 자율 루프 or --reanalyze로 1차 조정안. 실험 변수 1개 바꿔 추적
Week 3: 2차 조정. 잘 먹히는 포맷 집중. 업로드 시간/해시태그 미세조정
Week 4: 최종 성과 측정 + 실험 결과 정리 + 의사결정 로그로 포폴 서사 작성
```

안전망: 최소 자율 루프(T)가 늦으면 `--reanalyze` 수동 실행으로 대체. 미용사는 안 막힘.

## E단계 — 워크플로우 vs 에이전트 비교 실험 (나중에 셋업)
date: 2026-06-13 (세션 35 추가)

> 목적: "에이전트로 다시 만들었다"를 "에이전트가 워크플로우보다 나은지 측정했다"로 바꾼다.
> 비교 없는 하네스는 못한 LangGraph. 엄밀한 비교가 붙으면 극소수만 만드는 연구물.
> 선결: T(최소 자율 루프)가 존재해야 함 + 재현용 성과 스냅샷 1개.

### 실험 설계
- 동일 입력: 크리에이터 프로필 1명 + 동일 성과 데이터 스냅샷(고정).
- 조건 A (워크플로우): 1장 고정 파이프라인 --reanalyze 실행.
- 조건 B (에이전트): 2장 도구호출 루프(read_performance / web_search / log_decision) 실행.
- 통제: 같은 모델 + 같은 knowledge + 같은 성과 스냅샷. 오케스트레이션만 다름(고정 경로 vs 자율 루프).

### 측정 지표
| 지표 | 수집 |
|------|------|
| 산출물 품질 | 워커와 동일 LLM 저지 루브릭(0~100) |
| 비용 | 토큰 수 / 달러 |
| 지연 | wall-clock + LLM 콜 수 |
| 의사결정 근거성 | 에이전트의 도구 사용이 워크플로우 고정경로보다 더 근거 있는 조정을 냈는가(정성 + provenance 로그) |

### 정직성 가드
- 두 조건의 의사결정을 decision_log에 provenance로 기록. 체리피킹 금지.
- "더 비싼데 안 나으면" 그 결과도 그대로 보고 — 워크플로우가 충분하다는 발견일 수 있음(그것도 가치).

### 산출
- 비교 리포트 1장 -> 포폴/블로그 "workflow vs agent, 같은 과제로 다시 재봤다" 서사의 데이터 근거.
