- G4 ceo_summary가 전략의 단일 깨지기 쉬운 깔때기(LLM 문자열) -> 깨지면 4개 오염.
- G5 도메인 지식이 암묵적으로 미용사 전용(사례·예시). 범용 천장.

## 2. 타깃 아키텍처 — AgentDirective seam

깨지기 쉬운 단일 ceo_summary 문자열을 **구조화된 per-agent 지시 패킷**으로 교체.

```
[지금]  _interpret_goal -> plan.md(per-agent 지시, 고아)
        _build_ceo_summary -> ceo_summary 문자열 1개(전부 동일, 취약)

[타깃]  _interpret_goal -> plan.md + context["에이전트_지시"]={agent:지시}
        _build_ceo_summary -> context["directives"]={agent: AgentDirective.to_prompt_text()}
            = CEO전략 + 그 에이전트 핵심지시 + 구조화 사용자 방향/피드백/성과
        build_context -> 각 워커에 자기 directive를 ceo_summary 키로 주입(없으면 일반 폴백)
        base_agent._scoped_payload -> ceo_summary 그대로 읽음(무변경)
```

**Phase 2 정렬:** AgentDirective가 곧 도구 호출 payload다. 워커는 이미
run(scoped_context) = 도구 시그니처. 루프 오케스트레이터가 directive 만들어
워커-도구를 호출 -> 워커 재작성 없이 seam만 깔아둠. 선형 파이프라인은 같은 seam의
한 드라이버일 뿐.

## 3. Phase A — 완료 (구현 + 테스트)

- 신규 core/directive.py: AgentDirective(strategy/instruction/direction/feedback/
  performance) + to_prompt_text(). 방향은 DirectionProfile.to_prompt_text로 구조화
  주입, 채팅 append 방향은 verbatim 폴백, 블랭크/올-미정은 완전 생략(G4 보강).
- ceo.py: _parse_agent_instructions(plan 표 파싱) -> context["에이전트_지시"](G1 해소).
  _build_ceo_summary가 per-agent directives 생성. 일반 ceo_summary는 폴백 유지.
- agent_context.build_context: per-agent directive를 ceo_summary 키로 주입.
- 정리(G3): no-op _inject_summaries 제거(+호출부), _COMPRESS 주석 정정,
  agent_context "reserved" 주석 정정. (_compress_result는 브리핑이 소비 -> 보존)
- 테스트: tests/test_directive.py 8개. 전체 358 pass. CJK 0.

검증 포인트: CEO가 "경쟁분석은 X에 집중" 적으면 -> 경쟁분석 워커 directive에 도달.
