## 1. 도구 목록 (초안)

| 도구 | 시그니처(개념) | 재사용 | 권한 |
|------|---------------|--------|------|
| run_onboarding | (subject) -> 산출물 | 1장 워크플로우 전체 | 첫 분석 1패스 |

## 2. 루프 구조 (초안)

```
CEO.run(goal, subject):
  history = [system(목표+도구설명), user(subject + 현재 outputs/성과 상태)]
  for i in range(MAX_ITER):
      resp = call_llm_messages(history, tools=TOOLS)
      if resp.tool_calls:
          for call in resp.tool_calls:
              result = TOOLS[call.name](**call.args)   # 실행
              log_decision("AI", call, result)         # provenance
              history.append(tool_result(call, result)) # 관찰
          continue
      else:
          if "finish" in resp: break
          history.append(resp)
  guardrail_check()
```

자율성의 본체는 첫 분석이 아니라 **시간축 관리**: read_performance 결과를 보고
rerun_worker를 다른 focus로 부르는 판단. 첫 온보딩은 run_onboarding 도구 1회로 끝.

## 4. 검증 설계 (초안)

- 실데이터 기반(가상 아님): 컨시어지 운영 중 들어온 실제 성과로 루프 검증.
- 측정: (a) 도구 호출 실제 발생 (b) 경로가 관찰에 따라 변함 (c) 판단 타당(폴백 아님)
  (d) 루프당 콜 수/비용 (e) 정지조건 작동.
- baseline 비교: 1장 워크플로우(고정) vs 최소 자율 루프(가변) 품질/비용.
