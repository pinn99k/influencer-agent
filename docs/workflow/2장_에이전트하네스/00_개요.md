## 목표 재정의 (cross-verify 후 최종)

> 핵심 목표는 "멀티 에이전트 구현"이 아니라 **"실제 미용사 1명을 성장시키고
> 그 과정을 포트폴리오로 증명하는 것"**. 기술적 우아함보다 실사용 검증이 우선.

- 북극성: AI가 인플루언서를 **시간에 걸쳐** 관리한다(첫 분석이 아니라 시간축 경영에 자율성).
- 검증 방식: **컨시어지** — 핑구가 운영자로 1장 시스템을 돌리고, 미용사는 "서비스"를 받는다.
  미용사는 소프트웨어를 만지지 않는다(배포 불필요). 전략이 먹히는지를 먼저 증명.
- 자율 하네스: 실사용을 늦추는 우회로가 아니라, 실데이터 위에서 **최소 형태로** 구현.
  + 마케팅 성과가 안 나와도 "실데이터로 작동한 자율 에이전트"가 포폴 보험이 된다.

---

## 전달 모델 — 컨시어지 (확정)

```
핑구(운영자)가 로컬에서 python main.py 실행 -> 산출물 생성
  -> 미용사에겐 "전략 요약 + 이번 주 찍을 것 3개"만 전달(카톡/노션)
  -> 미용사는 콘텐츠만 올림 -> 성과를 핑구에게 보고
  -> 핑구가 --reanalyze(점차 최소 자율 루프) 실행 -> 수정안 전달
```

- 미용사 경험 = 소프트웨어가 아니라 **관리 서비스**.
- 1장(현재) 만으로 **이번 주 시작 가능**. 배포는 OUT(확장 단계로 이월).

---

## 2축 성공 기준 (cross-verify 보정)

- **실사용 축:** 팔로워 0 -> 300+ (최소) / 500+ (목표), 콘텐츠 12개+.
- **포폴 축:** 측정 가능한 실험 설계 + 의사결정/provenance 로그 +
  최소 자율 에이전트가 실데이터로 실제 조정한 기록.
  -> 팔로워가 안 늘어도 "왜 안 늘었는지 데이터로 설명 가능"하면 부분 성공.
- **실패:** 에이전트 95점 + 팔로워 30명 + 이유 설명 불가.

## 우선순위 (이 순서가 로드맵을 가른다)

| 순위 | 항목 | 비고 |
|:---:|------|------|
| 1 | 측정·실험 설계 시스템 | KPI + 콘텐츠 변수 태깅 + 성과 귀속 + 의사결정/provenance 로그 |
| 2 | 컨시어지 온보딩 배포 | 실제 미용사 적용, 4주 시계 시작 (1장 그대로) |
| 3 | 최소 자율 루프 | 성과 읽기 -> 분석 -> 개선안. 2주차부터. 진짜 도구+루프. 포폴 보험 |
| 4 | CEO 판단 폴백 가시화 (트랙 0) | 3번에 흡수 — 자율 루프 전에 판단 신뢰성 확보 |

## 범위 (IN)

- M. 측정·실험 레이어 (KPI/태깅/귀속/의사결정 로그/provenance)
- D. 컨시어지 온보딩 운영 (실제 미용사 1명 + 4주 운영)
- T. 최소 자율 루프 (도구 호출 + 루프, CEO 단일 에이전트)
- V. 트랙 0 (폴백 가시화)

## 범위 (OUT)

- 웹 배포 / 인증 / 멀티 사용자 — 확장 단계 (컨시어지로 검증 후)
- 완전 자율 관리 시스템 — 최소 루프만. 욕심내면 4주 초과
- 진짜 멀티 에이전트(워커별 루프) — 단일 CEO + 워커는 도구
- 자기개선(코드/프롬프트 자가수정) — outputs/ 밖 쓰기는 다음 장
- Claude Agent SDK / LangChain

## 선결 조건

1. **측정 체계 먼저** — 데이터가 아니라 "측정 가능한 실험 설계"가 검증을 만든다.
   콘텐츠 변수(주제/빈도/시간대/형식)를 태깅해 성과를 귀속할 수 없으면 노이즈일 뿐.
2. **트랙 0** — 자율 루프 + 미검증 판단 + 조용한 폴백 = 증폭된 쓰레기. 폴백 가시화 먼저.
