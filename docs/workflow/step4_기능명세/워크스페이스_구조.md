# 워크스페이스 폴더 구조
updated: 2026-05-12

> 이 문서는 실제 구현 시 파일 저장 위치의 기준이다.
> 변경 시 ADR 작성 + CLAUDE.md Structure 섹션 동시 업데이트 필수.

---

## 핵심 원칙

| 레이어 | 위치 | 대상 | 수정 주체 |
|--------|------|------|-----------|
| 사용자 산출물 | `outputs/{인플루언서명}/산출물/` | 핑구가 읽고 직접 작업 | 핑구 + CEO |
| 인수인계 | `outputs/{인플루언서명}/인수인계/` | 핑구가 읽음 | CEO |
| CEO 관리 | `outputs/{인플루언서명}/.system/ceo/` | 전략·상태 추적 | CEO |
| 에이전트 작업 | `outputs/{인플루언서명}/.system/agents/` | 중간 결과·검증·재작업 | 각 에이전트 |
| 보고 이력 | `outputs/{인플루언서명}/.system/briefings/` | 회장 보고 기록 | CEO |
| 프롬프트 | `outputs/{인플루언서명}/.system/prompts/` | Spiral 0-A 전용 | CEO |
| 로그 | `outputs/{인플루언서명}/.system/logs/` | UI 실시간 표시 | 시스템 자동 |

```

    │   └── 📁 .versions/                       ← 덮어쓰기 전 자동 보관
    │       ├── 01_대상분석_v1.md
```

## 버전 관리 규칙

- 파일 덮어쓰기 전 → `.versions/`에 `_{v숫자}` suffix로 자동 복사 후 덮어씀
- 버전 번호는 날짜 아님 — 실행 횟수 기준 (v1, v2, v3...)
- `.versions/` 파일은 읽기 전용 취급 — 직접 수정 금지
- 버전 관리 대상: 산출물/ 전체 + .system/agents/ raw_output.md
- 버전 관리 제외: validation.md, rework.md, state.md, plan.md (snapshots/로 대체)

## 파일별 저장 규칙

| 파일 | 저장 시점 | 덮어쓰기 | 버전 보관 |
|------|-----------|----------|-----------|
| 산출물/0N_*.md | CEO OK 판정 후 | ✅ (이전 버전 .versions/ 이동) | ✅ |
| 산출물/최종리포트.md | 전체 완료 후 | ✅ | ✅ |
| .system/agents/raw_output.md | 에이전트 결과 수신 즉시 | ✅ (이전 버전 .versions/ 이동) | ✅ |
| .system/agents/validation.md | 검증기 실행 후 | ✅ | ❌ |
| .system/agents/rework.md | 재작업 지시 시 | ✅ | ❌ |
| .system/ceo/plan.md | 전략 수립 후 + 판단 루프마다 | ✅ | snapshots/ |
| .system/ceo/state.md | 에이전트 호출 전·후 | ✅ | snapshots/ |
| .system/briefings/*.md | 회장 보고 발생 시 | ❌ (날짜+조건번호로 구분) | — |
| 인수인계/*.md | 세션 종료 시 | ❌ (날짜로 구분) | — |
| .system/logs/*.jsonl | 실행 이벤트마다 append | append | — |

## Spiral별 폴더 사용 현황

| Spiral | 사용 폴더 |
|--------|-----------|
| 0-A | `.system/prompts/` 만 생성 |
| 0-B | `.system/ceo/`, `.system/agents/`, `산출물/`, `인수인계/` |
| 1 | 전체 (`.system/agents/` 에이전트별 순서대로) |
| 2 | `.system/briefings/` 추가 |
| 3 | `.system/logs/` 추가 (UI 연동) |
