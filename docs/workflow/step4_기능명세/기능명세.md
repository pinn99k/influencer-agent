# Step 4 — 기능명세 (에이전트 설계 기준)
updated: 2026-05-12

> 이 문서는 구현 내내 변하지 않는 설계 기준이다.
> 변경 필요 시 → ADR 작성 후 핑구 승인 → 수정.
>
> **이 문서는 지도다.** 요약 + 참조 링크만 담는다.
> 상세 설계 → CEO/ 및 하위에이전트/ 폴더 참조.

---

## 에이전트 구조 요약

```
핑구 (회장)
  ↓ 목적 + 인풋 7개 필드
CEO 에이전트 (자율 경영자)
  ├─ [1] 목적 분석
  ├─ [2] 전략 수립 → 실행 계획 파일 저장
  ├─ [3] 하위 에이전트 역할 할당 + 컨텍스트 전달
  ├─ [4] 산출물 수신 → 품질 판단 → 피드백 or 승인
  │       NG → 피드백 + 재작업 요청 (max 1회)
  │       재작업도 NG → 회장 보고
  ├─ [5] 다음 단계 판단 (상태 파일 기반)
  └─ [6] 최종 판단 → 리포트 저장 → 회장 보고
    ↓ (하위 에이전트는 CEO 지시에만 응답)
대상 분석 ← 경쟁 분석 ← 플랫폼 추천 ← 컨셉 기획
(순서는 CEO가 판단. MVP 기본값: 대상→경쟁→플랫폼→컨셉)
```

---

## 1. CEO 에이전트 (자율 경영자)

> 상세 설계 → `CEO/` 폴더 참조

| 파일 | 내용 |
|------|------|
| [01_역할정의.md](CEO/01_역할정의.md) | CEO 철학, 하는 것/하지 않는 것 |
| [02_판단루프.md](CEO/02_판단루프.md) | 검증기→CEO판단→OK/NG 분기, 재작업 지시 형식 |
| [03_전략수립.md](CEO/03_전략수립.md) | 목적 분석 4문답, plan.md 출력 형식, 재질문 트리거 |
| [04_상태관리.md](CEO/04_상태관리.md) | state.md 구조, 트리거별 업데이트 시점 |
| [05_컨텍스트관리.md](CEO/05_컨텍스트관리.md) | 컨텍스트 키 표준, 에이전트별 전달 규칙 |
| [06_품질판단.md](CEO/06_품질판단.md) | 판단 기준, 피드백 프로토콜 |
| [07_회장보고.md](CEO/07_회장보고.md) | 보고 조건 10개 상세, 브리핑 형식 |
| [08_제약사항.md](CEO/08_제약사항.md) | CONSTRAINTS 전체, 금지 패턴 |

### 입력
- 핑구: 목적 + 대상자 7개 필드 (이름, 직업, 특기, 성격, 타겟연령대, SNS경험, 목표)
- 핑구: 레퍼런스 (선택 — 영상/이미지 URL 또는 파일 경로, 없으면 생략)

> 레퍼런스는 CEO가 대신 제공 불가. 핑구만 줄 수 있는 인풋. → ADR 008 참조

### 핑구 작업 요청 지점 (CEO가 멈추고 핑구에게 실행을 넘기는 시점)

| 시점 | 요청 내용 | 필수 여부 |
|------|-----------|-----------|
| Spiral 0-A 완료 후 | 생성된 프롬프트를 Claude/GPT에 직접 붙여넣어 품질 확인 | 필수 |
| 컨셉 기획 에이전트 실행 전 | 레퍼런스 영상/이미지 제공 | 선택 |
| 최종 리포트 승인 후 | 실제 인플루언서에게 결과 전달 + 적용 | 필수 |

> 핑구 작업 요청 ≠ 회장 보고. 판단 요청이 아닌 실행 요청. → ADR 008 참조

### 출력
- 실행 계획 파일 (`plan.md`)
- 상태 파일 (`state.md`, 상시 갱신)
- 하위 에이전트별 산출물 파일
- 최종 리포트 (`최종리포트.md`)
- 완료 보고 (CLI 출력)

## 2. 대상 분석 에이전트

> 상세 설계 → `하위에이전트/01_대상분석.md` 참조

인풋 7개 필드를 기반으로 강점 3개, 약점 3개, 차별점 1개를 분석한다.
각 항목마다 입력 필드 2개 이상을 근거로 연결. 추측 금지.

---

## 3. 경쟁 분석 에이전트

> 상세 설계 → `하위에이전트/02_경쟁분석.md` 참조

Serper 검색 결과 + 대상 분석 결과를 기반으로 유사 포지션 3개, 공백 기회, 차별 전략을 도출한다.
검색 결과 없을 시 fallback: LLM 추론 + "[데이터 부족 — 추론 기반]" 명시.

---

## 4. 플랫폼 추천 에이전트

> 상세 설계 → `하위에이전트/03_플랫폼추천.md` 참조

대상자 정보 + 대상 분석 + 경쟁 분석을 기반으로 1순위·2순위 플랫폼과 비추천 플랫폼 1개를 도출한다.
근거 없는 추천 금지. 3개 이상 추천 금지.

---

## 5. 컨셉 기획 에이전트

> 상세 설계 → `하위에이전트/04_컨셉기획.md` 참조

누적 컨텍스트(대상 분석·경쟁 분석·플랫폼 추천) 전체를 기반으로 콘텐츠 컨셉 3개와 영상 아이디어 5개를 생성한다.
대상자 성격(말하기 스타일) 반영 필수. 컨셉 3개 간 차별성 필수.
