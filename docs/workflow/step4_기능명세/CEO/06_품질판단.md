## CEO 주관 판단 체크리스트

검증기 PASS 후 CEO가 아래 3가지를 순서대로 확인한다.

```
1. 다음 에이전트가 이 결과로 작업할 수 있는가?
   → 구체적 수치·근거가 있는가? 아니면 공허한 일반론인가?

2. 프로젝트 목적(핑구가 제시한 목표)에 부합하는가?
   → 대상자 특성이 반영되어 있는가? 누구에게나 해당되는 말인가?

3. 대상자 입력 필드가 실제로 연결되어 있는가?
   → 근거 없는 단정이 있는가? "정보 없음" 처리가 제대로 되었는가?
```

3가지 모두 YES → OK
하나라도 NO → NG (회장 보고 조건 확인 후 재작업 지시)

## Generic 판정 기준

Generic = "이 대상자가 아닌 다른 누구에게도 해당되는 말"

### 판정 방법

결과 문장에서 대상자 이름을 지우고 읽었을 때 — 여전히 말이 되면 generic이다.

### 나쁜 예시 vs 좋은 예시

**대상 분석 — 강점**

| ❌ Generic | ✅ Specific |
|-----------|------------|
| "꾸준한 업로드로 구독자를 늘릴 수 있다" | "미용사 경력 7년의 실무 기술을 단계별로 보여주는 콘텐츠는 취준생 타겟에게 차별점이 된다" |
| "친근한 성격이 장점이다" | "ENFP 성격 + 20대 타겟 → 공감형 리액션 콘텐츠에 자연스럽게 맞는다" |

**경쟁 분석 — 공백 기회**

| ❌ Generic | ✅ Specific |
|-----------|------------|
| "아직 블루오션이 많다" | "검색 결과 기준, 미용사 출신 유튜버 중 취업 실무 콘텐츠를 다루는 채널이 없다" |

**컨셉 기획 — 영상 아이디어**

| ❌ Generic | ✅ Specific |
|-----------|------------|
| "브이로그 콘텐츠를 만들어보세요" | "훅: '미용학원에서 절대 안 가르쳐주는 것' / 구성: 실수 장면→교정→완성 / CTA: 저장 유도" |

## 에이전트별 CEO 판단 포인트

각 에이전트마다 CEO가 특히 집중해서 봐야 하는 항목.

| 에이전트 | 핵심 판단 포인트 |
|---------|----------------|
| 대상 분석 | 강점·약점이 입력 필드와 실제로 연결되어 있는가 (근거 필드 2개 이상) |
| 경쟁 분석 | Serper 결과 또는 fallback 명시 여부 / 공백 기회가 대상 분석과 연결되는가 |
| 플랫폼 추천 | 비추천 플랫폼 포함 여부 / 추천 이유가 대상자 데이터 기반인가 |
| 컨셉 기획 | 컨셉 3개 간 차별성 / 대상자 성격(말하기 스타일) 실제 반영 여부 |

## NG 피드백 작성 원칙

피드백은 "무엇이 왜 문제인지 + 어떻게 고쳐야 하는지"를 모두 포함해야 한다.
형식 → `02_판단루프.md` 재작업 지시 형식 참조.

**금지 피드백 패턴:**
- "더 구체적으로 써줘" → 무엇을 구체적으로?
- "대상자를 더 반영해줘" → 어떤 필드가 어떻게 빠졌는가?
- "다시 해줘" → 재작업 지시 아님, 단순 재호출

근거: 구체적 수정 지시 없는 재호출은 같은 결과를 반복한다. (AIEngineering.md 04_HARNESS)
