# CEO 에이전트 — 회장 보고
updated: 2026-05-12

---

## 한 줄 정의

> CEO가 단독 결정 불가 또는 핑구 실행이 필요한 시점에 작업을 멈추고 핑구에게 넘기는 프로토콜

---

## 보고 유형 5종

| 유형 | 의미 | 핑구에게 필요한 행동 |
|------|------|---------------------|
| 선택 | CEO가 A/B 방향 결정 불가 | 선택지 중 고름 |
| 승인 | 진행 전 핑구 확인 필요 | 승인 또는 거부 |
| 정보제공 | CEO 판단에 핑구 인풋 부족 | 추가 정보 제공 |
| 중단보고 | 계속 진행 불가 | 방향 결정 |
| 작업요청 | CEO가 실행 불가 — 핑구만 할 수 있음 | 직접 수행 후 결과 전달 |

> 작업요청 ≠ 나머지 4종. 판단 요청이 아닌 실행 요청. → ADR 008 참조

## 회장 보고 조건 10개

| # | 조건 | 유형 | 트리거 |
|---|------|------|--------|
| 1 | 비용 발생하는 모든 결정 (API 교체, 유료 툴 등) | 승인 | CEO 판단 |
| 2 | 법적 리스크 감지 (실인물 명예훼손, 개인정보 포함 출력) | 중단보고 | CEO 판단 |
| 3 | 초기 목표 변경 필요 시 | 선택 | CEO 판단 |
| 4 | 크리에이터 교체·전면 수정 등 사람에게 영향 | 선택 | CEO 판단 |
| 5 | 에이전트 2회 재작업 후도 실패 | 선택 | 자동 (MAX_RETRY 초과) |
| 6 | 에이전트 간 결과 충돌 | 선택 | CEO 판단 |
| 7 | 실제 외부 사람 컨택 필요 시 | 승인 | CEO 판단 |
| 8 | 개인정보·결과 외부 공개 필요 시 | 승인 | CEO 판단 |
| 9 | AI 산출물이 실제 실행으로 넘어가는 시점 | 승인 | 자동 (최종 리포트 저장 직전) |
| 10 | 회장 기대치와 결과가 크게 다를 것 같을 때 | 정보제공 | CEO 판단 |

## 핑구 작업 요청 지점 (MVP 기준)

보고 조건과 별개. CEO가 제공 불가한 인풋·실행이 필요한 시점.

| 시점 | 요청 내용 | 필수 여부 |
|------|-----------|-----------|
| Spiral 0-A 완료 후 | 프롬프트 파일을 Claude/GPT에 직접 붙여넣어 품질 확인 | 필수 |
| 컨셉 기획 에이전트 실행 전 | 레퍼런스 영상/이미지 제공 | 선택 |
| 최종 리포트 승인 후 | 실제 인플루언서에게 결과 전달 + 적용 | 필수 |

---

## 즉시 중단 규칙

보고 조건 감지 순간 — 에이전트 호출 중단, 재작업 금지, 즉시 브리핑 생성.

```
감지 → state.md current_phase = "회장보고대기" 업데이트
     → 브리핑 파일 생성 (.system/briefings/)
     → 핑구에게 출력
     → 핑구 결정 수신 전까지 모든 에이전트 호출 중단
```

## 브리핑 형식 — 보고 조건 (유형별)

### 선택 / 중단보고 / 정보제공

```
## 회장 보고
보고 유형: {선택 | 중단보고 | 정보제공}
보고 조건: {조건 번호} — {조건 내용}
현재 상황: {무슨 일이 발생했는가 — 구체적으로}
선택지:
  A. {선택지 A} — 예상 결과: {결과}
  B. {선택지 B} — 예상 결과: {결과}
권고안: {CEO 판단 — 근거 포함}
결정 요청: A 또는 B 선택 후 진행
```

### 승인

```
## 회장 보고
보고 유형: 승인
보고 조건: {조건 번호} — {조건 내용}
현재 상황: {무슨 일인가}
진행하려는 행동: {CEO가 다음에 하려는 것}
승인 요청: 진행 승인 / 중단 중 선택
```

### 정보제공

```
## 회장 보고
보고 유형: 정보제공
보고 조건: 10 — 기대치와 결과 괴리 감지
현재 상황: {어떤 산출물이 어떤 방향으로 나왔는가}
괴리 지점: {기대했던 것 vs 실제 나온 것}
필요한 정보: {핑구에게 필요한 인풋 — 구체적으로}
```

```
## 핑구 작업 요청
요청 유형: 작업 요청

완료 후 전달 방법: {결과를 어떻게 CEO에게 돌려주는지}
선택 여부: 필수 / 선택
```

## 브리핑 파일 저장 규칙

- 위치: `.system/briefings/briefing_{datetime}_{조건번호}.md`
- 덮어쓰기 금지 — 날짜+조건번호로 구분
- 핑구 결정 수신 후 → 해당 파일에 결정 내용 append
