# CEO 에이전트 — 상태 관리
updated: 2026-05-12

---

## 한 줄 정의

> CEO가 세션 전 과정에서 실행 상태를 파일로 추적하는 방식

---

## plan.md vs state.md 역할 구분

| 파일 | 역할 | 질문 |
|------|------|------|
| plan.md | 전략 + 실행 계획 (목적 중심) | "왜 이 순서로 하는가?" |
| state.md | 현재 실행 상태 (진행 추적) | "지금 어디까지 왔는가?" |

둘 다 `.system/ceo/`에 위치. 세션 종료 시 `snapshots/`에 날짜별 복사 보관.
파일 저장 위치 상세 → `워크스페이스_구조.md` 참조.

---

## state.md 구조

세션 시작 시 생성. 에이전트 호출 전·후 반드시 업데이트.

```markdown
# state.md
updated: {datetime}
대상자: {이름}
세션: {세션번호}

## 현재 상태
current_agent: {실행 중인 에이전트명 | "없음"}
current_phase: {전략수립 | 실행중 | 재작업 | 회장보고대기 | 완료}
retry_count: {0 | 1}  ← 현재 에이전트 재작업 횟수

## 에이전트 실행 이력
| 에이전트    | 상태 | 시작시간 | 완료시간 | 재작업 |
|-------------|------|----------|----------|--------|
| 01_대상분석 | DONE   | 14:01    | 14:03    | 없음   |
| 02_경쟁분석 | RUNNING   | 14:05    | -        | -      |
| 03_플랫폼추천 | PENDING  | -        | -        | -      |
| 04_컨셉기획 | PENDING   | -        | -        | -      |

## 회장 보고 이력
| 시점  | 조건번호 | 요약          | 처리결과    |
|-------|----------|---------------|-------------|
| 14:10 | 5번      | 대상분석 2회 실패 | 대기중   |

## 열린 이슈
- {재작업 중인 항목 / 미해결 문제 / 없으면 "없음"}
```

## 트리거별 업데이트 규칙

에이전트 호출 전 업데이트 → 호출 → 결과 수신 후 업데이트. 순서 어기지 않는다.

| 트리거 | 변경 필드 |
|--------|-----------|
| 전략 수립 완료 | `current_phase = 실행중`, 이력 테이블 초기화 |
| 에이전트 호출 **직전** | `current_agent = {에이전트명}`, 해당 행 `상태 = RUNNING`, `시작시간` |
| 검증기 PASS + CEO OK | 해당 행 `상태 = DONE`, `완료시간`, `current_agent = 없음` |
| NG → 재작업 지시 | `current_phase = 재작업`, `retry_count = 1`, 해당 행 `재작업 = 1회차` |
| 재작업 후 OK | 해당 행 `상태 = DONE`, `완료시간`, `재작업 = 완료`, `current_phase = 실행중`, `retry_count = 0` |
| 재작업 후도 NG | `current_phase = 회장보고대기`, 보고 이력 추가 |
| 회장 보고 발생 | `current_phase = 회장보고대기`, 보고 이력 행 추가 |
| 회장 결정 수신 | `current_phase = {결정에 따라}`, 보고 이력 `처리결과` 업데이트 |
| 전체 완료 | `current_phase = 완료`, `current_agent = 없음` |

근거: 호출 전 업데이트를 빠뜨리면 세션이 끊겼을 때 해당 에이전트가 실행 중인지
실행 전인지 판단할 수 없다. 반드시 호출 전·후 양쪽 다 업데이트한다.

## current_phase 상태 전이

```
전략수립
   │
   ▼
실행중 ──────────────────────────────────────────┐
   │                                              │
   │  NG (보고 조건 미해당)                        │
   ▼                                              │
재작업                                            │
   │                                              │
   ├── OK → 실행중 (retry_count 초기화)           │
   │                                              │
   └── NG (2회 실패) → 회장보고대기               │
                           │                      │
                           ▼                      │
                      회장 결정                   │
                           │                      │
                           └── 실행중 재개 ───────┘
                                                  │
                                                  ▼
                                               완료
```

회장보고대기 → 회장 결정 전까지 에이전트 호출 완전 중단.

## 구현 시 Python 구조 참고

```python
state = {
    "updated": "2026-05-12T14:05:00",
    "대상자": "홍길동",
    "세션": 1,
    "current_agent": "02_경쟁분석",
    "current_phase": "실행중",  # 전략수립|실행중|재작업|회장보고대기|완료
    "retry_count": 0,
    "agents": {
        "01_대상분석":  {"status": "done", "started": "14:01", "finished": "14:03", "rework": False},
        "02_경쟁분석":  {"status": "running", "started": "14:05", "finished": None, "rework": False},
        "03_플랫폼추천": {"status": "pending", "started": None, "finished": None, "rework": False},
        "04_컨셉기획":  {"status": "pending", "started": None, "finished": None, "rework": False},
    },
    "briefings": [],   # {"time": ..., "condition": 5, "summary": ..., "result": "대기중"}
    "open_issues": [],
}
```

state dict → state.md 변환은 CEO가 담당. 사람이 읽을 수 있는 마크다운으로 저장.
