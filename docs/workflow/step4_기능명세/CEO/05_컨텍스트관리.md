# CEO 에이전트 — 컨텍스트 관리
updated: 2026-05-12

---

## 한 줄 정의

> CEO가 하위 에이전트 결과를 누적하고, 각 에이전트에 필요한 키만 골라 전달하는 방식

---

## 컨텍스트 키 표준

```python
context = {
    "대상자": {
        "이름": str,
        "직업": str,
        "특기": str,
        "성격": str,
        "타겟연령대": str,
        "SNS경험": str,
        "목표": str,
    },
    "대상_분석":   str,   # "## 대상 분석 결과\n..." (에이전트 원본 출력)
    "경쟁_분석":   str,   # "## 경쟁 분석 결과\n..."
    "플랫폼_추천": str,   # "## 플랫폼 추천 결과\n..."
    "컨셉_기획":   str,   # "## 컨셉 기획 결과\n..."
    "검증_결과": {
        "단계": str,       # 에이전트명
        "통과여부": bool,
        "실패규칙": str,   # 없으면 None
        "재시도여부": bool,
    },
    "보고_조건": {
        "조건번호": int,
        "근거": str,
        "조치": str,       # "즉시중단" | "대기중" | "처리완료"
    },
}
```

빈 필드는 `None`으로 초기화. 공백 문자열 금지.

---

## 에이전트별 전달 규칙

각 에이전트 호출 시 필요한 키만 전달. Full 컨텍스트 통째로 넘기지 않는다.

| 에이전트 | 전달 키 | 전달하지 않는 키 |
|---------|---------|-----------------|
| 대상 분석 | `대상자` | 나머지 전부 |
| 경쟁 분석 | `대상자`, `대상_분석` | 플랫폼·컨셉·검증·보고 |
| 플랫폼 추천 | `대상자`, `대상_분석`, `경쟁_분석` | 컨셉·검증·보고 |
| 컨셉 기획 | `대상자`, `대상_분석`, `경쟁_분석`, `플랫폼_추천` | 검증·보고 |

근거: 필요 없는 키를 넘기면 토큰 낭비 + 에이전트가 엉뚱한 정보에 반응할 위험.

## 누적 방식

```python
# OK 판정 후 즉시 누적 (NG 상태에서 누적 금지)
def update_context(context, agent_name, result):
    # 키 = display_name (agent.display_name) — 아키텍처.md AGENT_KEY_MAP과 동일 기준
    key_map = {
        "대상분석":  "대상_분석",
        "경쟁분석":  "경쟁_분석",
        "플랫폼추천": "플랫폼_추천",
        "컨셉기획":  "컨셉_기획",
    }
    context[key_map[agent_name]] = result
```

- NG 상태(재작업 중 / 회장보고대기) 에서는 context 업데이트 금지
- 재작업 후 OK → 기존 키 덮어쓰기 (버전 관리는 `.system/agents/.versions/`에서 담당)

## Structured Context 선택 근거

우리 프로젝트는 **Structured Context** 방식을 사용한다.

| 방식 | 설명 | 문제 |
|------|------|------|
| Full | 전체 context를 매번 통째로 전달 | 토큰 낭비 (80~90% 불필요) |
| Summarized | 이전 결과를 요약해서 전달 | 정보 손실 → 다음 에이전트 품질 저하 |
| **Structured** | 필요한 섹션만 골라 전달 | **토큰 절감 + 정보 손실 없음** |

컨셉 기획 에이전트 호출 시 Full 방식 대비 토큰 약 60~80% 절감 예상.

## 토큰 한계 초과 시 처리

MVP 범위에서 토큰 한계 초과 가능성은 낮으나, 발생 시:

1. `대상_분석` 섹션이 가장 길어지는 경우: 요약 요청 (CEO가 직접)
2. 요약 후에도 초과: 회장 보고 (조건 외 상황 — CEO 단독 판단 금지)

MVP에서 실제 발생 시 ADR 작성 후 처리 방식 확정.
