# 세션 19 리뷰 — 프롬프트 강화 + Department Layer + 하네스 엔지니어링
날짜: 2026-05-18
Spiral: 1 (품질 레이어 + E2E 검증)

---

## 목표
출력 품질 60/100 → 80+ 달성을 위한 프레임워크 강화.

## 결정사항

### 1. Department Layer 도입
- CEO와 에이전트 사이에 기획본부(PlanningDepartment) 추가
- 컨텍스트 압축: 이전 결과를 3줄 요약으로 압축 후 다음 에이전트 전달
- 전략 브리핑: CEO에게 정제된 브리핑 전달

### 2. 프롬프트 엔지니어링
- 4개 프롬프트에 [THINKING] + [SELF-CHECK] 섹션 추가
- [THINKING]: 단계별 사고 흐름 강제 (<think> 태그)
- [SELF-CHECK]: 검증 항목 체크리스트 (기능명세 + OutputValidator 기준)
- [EXAMPLES]: GOOD/BAD 예시로 품질 기준 명시
- 참조 문서: knowledge + refs + 기능명세 + 설계원칙 전체

### 3. 하네스 엔지니어링
- Generic phrase 감지: OutputValidator에 7개 금지 문구 추가
- Two-pass generation: 컨셉기획 에이전트만 draft → refine 2회 생성
- Foreign char regex 확장: CJK + 일본어 + accented Latin + Cyrillic + Arabic + Thai
- MAX_RETRY=1 복원 (품질 재시도 허용)

### 4. 테스트 확장
- 74 → 101 테스트 (27개 추가)
- 신규: foreign char 감지 6개, generic phrase 4개, department layer 9개, two-pass 2개, regex 6개
