## 자체 피드백

### 잘한 것
- 핑구 피드백 정확히 반영: "속도 최적화보다 사고 품질" → multi-step 분할

- Knowledge 파일 CJK 오염 없이 깔끔하게 생성

### 못한 것
- Gemini 503 대응 부재: fallback 없이 Gemini만 의존 → E2E 실패
- E2E 테스트를 세션 후반에 시도 → 시간 부족으로 검증 불가
- PowerShell here-string 내 Python dict 구문 오류 → 시간 낭비

### 교훈
- 외부 API 의존 기능은 fallback을 먼저 구현한 후 테스트할 것
- E2E는 세션 초반에 먼저 돌려서 baseline 확인할 것
- 복잡한 Python 코드는 항상 .py 파일로 작성 (lessons.md 규칙 재확인)
