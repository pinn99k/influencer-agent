## 잘된 것

- **자체 피드백 → 제안 → 승인 → 작성** 흐름이 정착됨. 방향 오류 없이 진행
- 워크스페이스 폴더 구조를 시스템 중심 → 사용자 중심으로 전환한 것 — 핵심 통찰
- STATUS.md 도입으로 세션 간 미완료 작업 추적 체계 생김
- 충돌 체크를 미리 해서 기능명세.md 문제를 다음 세션 전에 파악함

## 개선할 것

- 세션 초반에 충돌 체크를 했어야 함 — 파일 여러 개 작성 후 충돌 발견은 재작업 리스크
- 컨텍스트 키 표준을 기능명세.md에 중복 작성한 것은 사전에 막았어야 함
- 워크스페이스 구조 설계가 중반에 끼어들어 CEO/ 작성 흐름이 끊김 — 순서 판단 개선 필요
