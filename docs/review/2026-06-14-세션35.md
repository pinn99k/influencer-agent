  관리대시보드(KPI/4주캘린더/할일) -> 산출물4+최종리포트 열람 -> 매니저알림. 콘솔 에러 0.
- 발견 nit: ceo_summary에 영어단어 혼입(Appeal/VISUAL, 산출물 본문은 깨끗), 캘린더 ** 마크다운 노출.

### E. 컨셉 출력 심화 (실 사용자 예으니 피드백)
- 진단: 예으니가 재분석 2번 했는데도 피드백 미반영. 근본원인=프롬프트 OUTPUT FORMAT이 천장
  (해시태그 1줄/캡션 1개/편집 1줄) -> 구조에 담을 칸이 없어 단어만 바뀜.
- 수정: 해시태그 3층(니치/트렌드/대형)+각 이유, '### 영상 편집 컨셉' 신설(훅/전개/자막/컷),
  캡션 2개+(정보형/공감형), 날조·어그로·소재지시 금지 CONSTRAINT, '추천' framing.
- 검증: 예으니 실 이전분석+실 피드백으로 컨셉 재생성(실 gpt-4o) 5/5 PASS. OutputValidator
  +픽스처 갱신. 370 pass 무회귀.

## 결정/교훈
- 출력 품질의 천장은 '프롬프트 OUTPUT FORMAT의 구조'다. 피드백이 깊이를 원해도 구조에 칸이
  없으면 반영 불가 -> '도달 != 준수'의 새 변종(구조 천장). 구조부터 열어야 함.
- AgentDirective가 파이프라인/루프 양쪽 payload로 승격 -> 설계 정합 확인.
- 통과 != 검증: 자율성 '작동'은 통합으로 증명, '이득'(워크플로우 대비)은 미측정(E단계 A/B 필요).
