## 프로젝트 성공 정의

### 한 줄 정의
이 프로젝트의 성공 = 실제 미용사 1명에게 시스템을 적용하여 1개월 내 측정 가능한 인플루언서 성과를 만드는 것.

### 성공은 수치로 판단한다

| 등급 | 팔로워/구독자 | 콘텐츠 수 | 추가 조건 |
|------|-------------|----------|----------|
| 최소 성공 | 0 → 300+ | 12개+ | 시스템 재분석 1회 실행 |
| 목표 성공 | 0 → 500+ | 12개+ | 1개 이상 조회수 1만+ |
| 대성공 | 0 → 1,000+ | 16개+ | 협찬/문의 1건+ |

### 기간
적용 시작일로부터 30일.

## 시스템 갭 분석

### 현실 기획사가 하는 일 vs 현재 시스템

| # | 기획사 기능 | 설명 | 현재 시스템 | 상태 |
|---|-----------|------|-----------|------|
| 1 | 대상 분석 | 크리에이터 강점/약점/차별점 | 대상분석 에이전트 | 완료 |
| 2 | 시장 조사 | 경쟁자 분석, 시장 공백 | 경쟁분석 에이전트 | 품질 개선 필요 |
| 3 | 플랫폼 결정 | 1순위/2순위 추천 + 이유 | 플랫폼추천 에이전트 | 완료 |
| 4 | 컨셉 확정 | 3컨셉 + 영상 아이디어 | 컨셉기획 에이전트 | 완료 |
| 5 | 콘텐츠 캘린더 | 4주치 구체적 일정 | 없음 | P0 필수 |
| 6 | 실행 가이드 | 촬영 방법, 편집 팁, 포맷 | 없음 | P0 필수 |
| 7 | 업로드 최적화 | 해시태그, 캡션, 시간대 | 없음 | P0 필수 |
| 8 | 성과 추적 | 팔로워, 조회수, 참여율 기록 | 없음 | P0 필수 |
| 9 | 재분석/피벗 | 데이터 기반 전략 수정 | 없음 | P0 필수 |

핵심: 현재 시스템은 1~4번(전략 수립)만 수행. 5~9번(실행 지원 + 성과 루프)이 없다.
산출물을 받은 미용사가 "그래서 내일 뭘 찍지?"에 답을 못 주면 시스템은 실패다.

## 4주 실행 로드맵

### 전제
- 시스템 확장 P0 완료 상태
- 실제 미용사 대상자 확보 + 정보 수집 완료
- Gemini 품질 확인 완료

### 시스템 준비 (D-7 ~ D-1)
1. P0 5개 항목 구현
2. Gemini E2E 품질 확인
3. 가상 미용사로 확장 산출물 테스트 → 품질 검증
4. 실제 미용사 대상자 정보 상세 수집

### Week 1 (D+1 ~ D+7): 초기 분석 + 첫 콘텐츠

시스템:
- 대상자 입력 → CEO 실행 → 확장 산출물 생성
  (대상분석 + 경쟁분석 + 플랫폼추천 + 컨셉기획 포함 캘린더/가이드 + 최종리포트)
- 핑구 검토 → 회장 보고 조건 9 (실행 단계 진입) 승인

사람:
- SNS 계정 세팅 (프로필, 바이오 — 산출물 기반)
- 첫 콘텐츠 3~4개 제작/업로드
- 초기 반응 관찰

### Week 2 (D+8 ~ D+14): 데이터 수집 + 중간 재분석

사람:
- 주 3~4개 콘텐츠 유지
- 성과 데이터 기록 → outputs/{name}/성과기록.md

시스템 (2주차 말):
- 핑구가 --reanalyze 실행
- CEO가 성과 데이터 기반 판단:
  - 컨셉 반응 저조 → 컨셉기획 재실행
  - 경쟁 환경 변화 → 경쟁분석 재실행
  - 플랫폼 부적합 → 플랫폼추천 재실행
- 후반 2주 캘린더 수정본 생성

### Week 3 (D+15 ~ D+21): 최적화

사람:
- 수정된 전략 반영, 잘 먹히는 포맷 집중
- 업로드 시간대/해시태그 미세 조정

시스템:
- 필요 시 2차 재분석

### Week 4 (D+22 ~ D+30): 성과 측정 + 판단

사람:
- 최종 성과 수치 기록

시스템:
- CEO 재실행 → 1개월 성과 리포트 생성
- 목표 대비 달성률 + 다음 1개월 전략 제안
- 성공 여부 판정 (위 수치 기준표 기준)

## 미용사 특수 고려사항

시스템이 산출물 생성 시 반영해야 할 직업 특성:

| 항목 | 내용 | 시스템 반영 방법 |
|------|------|----------------|
| 시간 제약 | 하루 8~10시간 근무, 콘텐츠 제작 시간 제한 | 캘린더: 주 2~4개, 각 30초 이내 기본 |
| 촬영 환경 | 살롱 자체가 세트 (장점) | 실행 가이드: 추가 장비 최소화 |
| 카메라 익숙도 | 내향적이면 말하기 부담 | 성격 기반 포맷 추천 (손만 나오는 시술 우선) |
| 고객 동의 | 시술 촬영 시 고객 동의 필요 | 회장 보고 조건 8 (개인정보 외부 공개) 연결 |
| 시각적 강점 | before/after, 컬러 변신 등 | 플랫폼: 인스타 릴스, 유튜브 쇼츠 우선 |

## CEO 재분석 루프 설계

### 현재 (1회 실행)
입력 → CEO → 에이전트 4개 순차 실행 → 산출물 → 끝

### 목표 (재분석 사이클)
초기 실행 → 산출물 + 캘린더 생성
    → 콘텐츠 제작/업로드 (사람)
    → 성과 기록 (outputs/{name}/성과기록.md)
    → --reanalyze 실행 (핑구 트리거)
    → CEO가 판단: 어떤 에이전트를 재실행할지
    → 필요한 에이전트만 재실행
    → 수정된 산출물 + 후반 캘린더
    → 반복

완전 자동 루프는 Phase 2+. MVP 확장에서는 핑구 수동 트리거.

### 재분석 시 CEO 컨텍스트에 추가되는 것
- 기존 산출물 4개 (이전 전략)
- 성과기록.md (수치 데이터)
- 핑구 피드백 (있으면)

## 리스크 대응

| 리스크 | 확률 | 영향 | 대응 |
|--------|------|------|------|
| 미용사가 콘텐츠 제작을 지속 못 함 | 높음 | 치명적 | 캘린더를 주 2개, 각 15초까지 유연 설계 |
| 경쟁분석이 부정확해서 잘못된 포지셔닝 | 중간 | 높음 | 2주차 재분석에서 실제 반응 데이터로 보정 |
| 한 달로는 의미 있는 수치가 안 나옴 | 중간 | 중간 | 최소 성공 기준을 팔로워 300으로 설정 |
| 산출물 품질이 실행 가이드로 부족 | 중간 | 높음 | 프롬프트 강화 후 테스트 1회 → 피드백 반영 후 본실행 |
| 플랫폼 알고리즘 변화 | 낮음 | 중간 | 멀티 플랫폼 전략 (1순위 + 2순위 동시 운영) |
