## 재시도 가치 판단

재작업 지시를 내리기 전 아래를 판단한다.

### 재작업할 가치가 있는 경우
- 실패 원인이 명확하고 수정 지시가 구체적으로 가능한 경우
- 해당 에이전트 결과 없이 다음 단계 진행이 불가능한 경우
- 재작업 비용(토큰 1회)이 전체 실행 대비 낮은 경우

### 에스컬레이션이 나은 경우
- 실패 원인이 에이전트 역량 한계(모델 품질 문제)인 경우
- 이미 1회 재작업했고 여전히 실패한 경우 → 조건 5
- 문제가 에이전트 외부(컨텍스트 부족, 목표 불명확)인 경우

## 모델 전환 판단

Groq llama3-70b로 품질이 지속 NG인 경우 모델 전환을 검토한다.
단, 모델 전환은 비용 발생 → **반드시 회장(핑구) 보고 후 승인**.

판단 기준:
- 같은 에이전트가 2회 이상 연속 NG → 모델 문제 가능성
- 재작업 지시가 구체적이었음에도 NG → 모델 역량 한계
- 특정 에이전트(예: 컨셉기획)만 NG → 해당 에이전트만 교체

전체 모델 교체 금지. 병목 에이전트 단위로만 교체.

## 자원 배분 우선순위

한정된 자원 안에서 아래 순서로 우선 투자한다.

1. **완료 조건과 직결된 작업** — 건너뛸 수 없는 것
2. **다음 에이전트 인풋이 되는 결과** — 블로킹 작업
3. **품질 개선** — 이미 통과했지만 더 나은 결과
4. **로깅·문서화** — 중요하지만 즉시성 낮음

MVP에서는 4번은 생략 가능. 1~3번 먼저.
