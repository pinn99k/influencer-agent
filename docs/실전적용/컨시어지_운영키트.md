# 컨시어지 운영 키트 (실전 적용 런북)
# 목적: 실제 미용사 1명에게 시스템을 적용해 1개월 운영하는 핑구(운영자)용 실행 안내서.
# 원칙: 이 문서만 보고 처음부터 끝까지 혼자 운영할 수 있어야 한다.
updated: 2026-06-14 (세션 36)

---

## 0. 한눈에 보는 흐름
온보딩(정보 수집) -> 1차 분석(시스템) -> 산출물 검수(루브릭) -> 크리에이터 전달
-> 4주 운영(제작/업로드/측정) -> 2주차 말 재분석 -> 4주차 성과 판정.

성공 기준(요약): 30일 안에 팔로워 300+(최소) / 500+(목표) / 1,000+(대성공).
상세 수치표: docs/refs/실전적용_로드맵.md.

---

## 1. 온보딩 — 미용사에게 받을 정보
1차 분석 품질은 여기서 결정된다. 인터뷰(웹/CLI)가 이 항목을 묻는다. 빠진 칸이 많을수록 산출물이 일반론이 된다.

### 필수 7
- 이름(가명 가능 — 작업 폴더명이 된다)
- 직업 / 경력 (예: 미용사 3년차, 미용 전공 학생)
- 특기 (예: 커트, 염색, 펌 중 가장 자신 있는 것)
- 성격 (내향/외향, 말하기 부담 여부 — 포맷 추천에 직결)
- 타겟 연령대 (주로 어필하고 싶은 층)
- SNS 경험 (현재 운영 계정/팔로워 수)
- 목표 (6개월~1년 내 달성하고 싶은 것 — 장기 목표로 기록됨)

### 확장 4 (실행 가이드 품질을 좌우 — 가능하면 꼭)
- 가용시간 (하루 콘텐츠에 쓸 수 있는 시간 — 캘린더 분량 조절)
- 촬영환경 (살롱/집/학교 등 — "어디서 찍나"가 가이드에 반영됨)
- 카메라경험 (얼굴 노출 가능 여부)
- 예산 (장비/편집앱 유료 여부)

### 방향 3 (사용자가 정하는 전략 — 선택)
- 콘텐츠방향 / 핵심목표 / 중점전략

### 미용사 특수 — 반드시 확인
- 영상 소재가 고정돼 있나? (과제 영상, 모델 작업 등) -> 있으면 시스템은 "새로 찍어라"가
  아니라 "가진 소재를 어떤 각도로" 추천한다. 온보딩에 적어둘 것.
- 고객/모델이 영상에 나오나? -> 나오면 촬영 전 동의 필요(개인정보 = 회장 보고 조건 8).
- 시각적 강점(비포애프터, 컬러 변신)을 살릴 수 있나? -> 인스타 릴스/쇼츠 우선.

## 2. 시스템 구동 (1차 분석)
모든 명령은 src/ 기준. API 키는 src/.env 에 있어야 한다.

### 방법 A — 웹 (권장, 미용사와 함께 보기 좋음)
1. `cd src && python -m uvicorn api.main:app --port 8000`
2. 브라우저 http://localhost:8000
3. "대화 인터뷰" 탭에서 위 1번 항목을 대화로 채운다 -> 요약 확인 -> 분석 시작.
4. 대시보드에서 4개 분석 + 캘린더 + 최종리포트 + 매니저 알림 확인.

### 방법 B — CLI
1. `cd src && python main.py`  (대화형 인터뷰가 기본)
2. 완료 후 outputs/{이름}/산출물/ 에 결과 생성.

산출물 위치: `src/outputs/{이름}/산출물/`
- 01_대상분석.md / 02_경쟁분석.md / 03_플랫폼추천.md / 04_컨셉기획.md / 최종리포트.md

---

## 3. 산출물 검수 — 전달 전 필수 (루브릭)
크리에이터에게 주기 전에 운영자가 직접 본다. 기준: docs/실전적용/실행가능성_루브릭.md

체크(하나라도 FAIL이면 재생성하거나 수정):
- [ ] 이름이 모든 파일에서 정확한가 (오타/타인 이름/축약 없음)
- [ ] "정보 없음" 같은 빈칸 표시가 노출되지 않았나
- [ ] 성공지표 팔로워가 현실값(300~500)인가, 큰 수치는 "장기 목표"로 분리됐나
- [ ] 영상 아이디어 -> 4주 캘린더 -> 리포트 로드맵이 같은 계획으로 이어지나
- [ ] 미용사 실제 환경/제약이 촬영 가이드에 반영됐나
- [ ] 미용사가 남긴 요구(있으면)가 결과에 실제로 반영됐나

검수 통과 = 실행 단계 진입(회장 보고 조건 9). 핑구 승인 후 전달.

---

## 4. 크리에이터에게 전달
가장 먼저 보여줄 것: **최종리포트.md**(한 줄 결론 + 지금 당장 할 3가지 + 4주 로드맵).
함께 줄 것: **04_컨셉기획.md**(4주 캘린더 + 편집 컨셉 + 촬영 가이드 + 해시태그/캡션).
나머지(01~03)는 배경 자료. 처음엔 안 줘도 된다.

전달 메시지 예: "이번 주는 1주차 3개만. 캘린더의 1번부터. 해시태그/캡션은 04 문서 그대로 붙여 쓰면 됩니다."

## 5. 4주 운영 시계
주차별 측정 기록은 웹 매니저 도크의 "측정 기록"(KPI 미니폼 + 게시물 태깅) 사용.

| 주차 | 미용사 | 핑구(운영자) |
|------|--------|-------------|
| Week 1 | 계정/프로필 세팅(산출물 기반), 콘텐츠 3~4개 업로드 | 전달, 초기 반응 관찰 |
| Week 2 | 주 3~4개 유지, 성과(팔로워/조회/저장) 기록 | 2주차 말 **재분석 실행**(아래 6번) |
| Week 3 | 수정 전략 반영, 잘 되는 포맷 집중 | 필요 시 2차 재분석, 시간대/해시태그 미세조정 |
| Week 4 | 최종 성과 수치 기록 | 성과 리포트 생성, 목표 대비 판정 |

리스크 대응: 미용사가 제작을 못 따라오면 캘린더를 주 2개/각 15초로 줄인다.

## 6. 재분석 (2주차 말, 그리고 필요할 때)
성과 데이터/피드백을 시스템에 넣고 다시 돌리면, CEO가 "무엇을 왜 다시 할지" 판단해 필요한
부분만 재생성한다(보통 컨셉기획 + 최종리포트).

준비: outputs/{이름}/ 에 성과기록/피드백/방향이 있어야 한다(웹 매니저 도크에서 입력하거나
피드백.md 직접 작성). 입력이 하나도 없으면 재분석은 안전하게 생략된다.

실행:
- 웹: 매니저 도크에서 성과/피드백 입력 -> "전달하고 재분석".
- CLI: `cd src && python main.py --reanalyze --name {이름}`

주의: 1차는 인터뷰로 전체 정보를 받아야 한다. 재분석은 기존 산출물 + 성과/피드백/방향만
참조한다(대상자 기본정보는 1차에서 확정).

재생성 후 3번(루브릭 검수)을 다시 한다. 기존 산출물은 산출물/.versions/ 에 자동 백업된다.

## 7. 성과 판정 (30일)
| 등급 | 팔로워 | 콘텐츠 | 추가 |
|------|--------|--------|------|
| 최소 성공 | 300+ | 12개+ | 재분석 1회 실행 |
| 목표 성공 | 500+ | 12개+ | 조회수 1만+ 1개 |
| 대성공 | 1,000+ | 16개+ | 협찬/문의 1건+ |

판정 후: 성공이면 2차 사이클(다음 1개월 전략), 미달이면 어느 변수가 약했는지 측정 기록으로 복기.
