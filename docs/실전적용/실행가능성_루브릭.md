# 실사용자 실행가능성 루브릭
# 목적: 산출물을 "테스트 통과"가 아니라 "실제 사용자가 그대로 실행 가능한가"로 채점한다.
# 배경: 예으니(실사용자) 산출물이 370 테스트/E2E 통과에도 실사용 불가였다.
#       구조 검증(섹션 존재)과 실행가능성은 다른 시험이다.
updated: 2026-06-14 (세션 36)

---

## 채점 원칙
- PASS = 신규 사용자가 산출물만 보고 "내일 무엇을 어떤 각도로 찍고 어떻게 올릴지" 안다.
- 아래 항목 중 하나라도 FAIL이면 산출물 전체 FAIL. 부분 점수 없음.
- OutputValidator(섹션/키워드/길이)는 전제 조건일 뿐, 통과해도 이 루브릭을 따로 본다.

## 1. 정확성 (신뢰를 깨지 않는가)
- [ ] 대상자 이름이 모든 산출물에서 정확하다 (변형/오타/타인 이름 혼입 없음).
- [ ] 플레이스홀더("정보 없음" 등)가 사용자 화면에 노출되지 않는다.
- [ ] KPI/목표 수치가 근거 있고 현실적이다 (신규 계정 30일: 팔로워 300 최소 / 500 목표,
      대상자 장기 목표는 별도 구분). 근거 없는 수치 날조 없음.

## 2. 일관성 (하나의 계획인가)
- [ ] 할 일 목록이 하나의 계획으로 수렴한다. 서로 다른 제목의 콘텐츠 목록이 2개 이상
      충돌하지 않는다 (영상 아이디어 = 무엇을, 캘린더 = 언제, 최종리포트 로드맵 = 캘린더 요약).
- [ ] 컨셉에서 아이디어, 캘린더, 리포트 로드맵까지 같은 주제 줄기로 이어진다.

## 3. 개인화 (이 사람 것인가)
- [ ] 대상자의 실제 제약 (가용시간, 촬영환경, 카메라경험, 예산)이 가이드에 반영됐다.
- [ ] 일반론(아무에게나 맞는 말)이 아니라 이 대상자 상황에 맞는 구체 지시다.

## 4. 사용자 피드백 충족 (말한 대로 했는가)
- [ ] 사용자가 글로 남긴 요구를 항목별로 충족했다 (섹션 존재가 아니라 요구 내용 기준).
- [ ] "추천만 / 소재 고정 / 어그로 빼기 / 이유 설명" 같은 지시가 결과에 실제로 반영됐다.

## 5. 실행가능성 (바로 할 수 있는가)
- [ ] 해시태그에 각 층별 구체 이유가 있다 (복붙 예시 문구가 아니라 이 주제 기준).
- [ ] 캡션 템플릿이 그대로 붙여쓸 수 있는 완성형 2개 이상이다.
- [ ] 영상 편집 컨셉이 훅/전개/자막/컷 4요소로 구체적이다.
- [ ] "지금 당장 할 3가지"가 3일 안에 실행 가능한 구체 행동이다.

---

## 적용 방법
1. 생성기 수정 후, 대상 사용자(예: 예으니)로 실제 재생성한다 (--reanalyze).
2. 위 5개 영역을 사람이 직접 체크한다 (자동 통과로 대체 금지).
3. 사용자가 남긴 피드백 원문과 1:1로 대조한다.
4. 하나라도 FAIL이면 생성기를 다시 고친다. 통과해야 "실전 적용 준비됨".
