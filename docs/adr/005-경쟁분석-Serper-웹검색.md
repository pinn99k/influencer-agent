# ADR 005 — 경쟁 분석 에이전트 웹 검색 방식 (Serper)
date: 2026-05-12
status: 확정

---

## 결정

경쟁 분석 에이전트의 입력 방식으로 **C방식 (Serper API 자동 검색)** 을 채택한다.
핑구가 추가 참고 계정을 보완 입력하는 것은 선택 사항으로 허용한다.

---

## 배경

경쟁 분석 에이전트는 "유사 크리에이터 포지셔닝 + 공백 기회" 분석이 목적이다.
Groq llama3-70b는 실시간 웹 데이터에 접근 불가 → 외부 검색 도구 필요.

---

## 대안 비교

| 방식 | 품질 | 핑구 개입 | 추가 도구 | 비용 |
|------|------|-----------|-----------|------|
| A. LLM 추측 | 낮음 (generic 위험) | 없음 | 없음 | 0 |
| B. 핑구 직접 입력 | 높음 | 있음 (매번 조사 필요) | 없음 | 0 |
| C. Serper 자동 검색 | 높음 | 없음 (선택적 보완) | Serper API | 무료 티어 |

---

## 선택 근거

- MVP 품질 기준 "타협 없음" → A 방식 탈락
- 핑구 1인 운영, 매번 수동 조사 = 자동화 목적 훼손 → B 방식 탈락
- Serper 무료 티어 2,500건/월 → MVP 규모에서 비용 0
- Groq이 검색 결과를 컨텍스트로 받아 분석 → 실제 데이터 기반 품질 확보

---

## 구현 방식

```
흐름:
대상자 정보 수신
  → Serper API로 "{직업} {특기} 유튜버/인스타" 검색
  → 상위 5개 결과 수집 (채널명, 구독자 수, 콘텐츠 유형)
  → Groq이 해당 데이터 기반으로 포지셔닝 분석
  → 핑구 추가 입력 있으면 병합 후 분석

출력:
  - 유사 포지션 요약 (3개)
  - 공백 기회
  - 차별 전략 제안
```

---

## 영향

- Stack 추가: `serper-python` (pip install)
- 환경변수 추가: `SERPER_API_KEY`
- 인풋 필드 추가 없음 (자동 검색)
- Spiral 1 (경쟁 분석 에이전트 구현) 시 적용

---

## 미결 사항

- Serper 검색 결과 파싱 안정성 → Spiral 1 구현 시 검증 필요
- 검색 결과 없을 경우 fallback → A방식(LLM 추측)으로 자동 전환
