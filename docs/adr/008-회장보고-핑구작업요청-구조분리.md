# ADR 008 — 회장 보고 조건 vs 핑구 작업 요청 구조 분리
date: 2026-05-12
status: 확정

---

## 배경

기존 설계는 핑구의 개입을 "회장 보고 조건 10개"로만 정의했다.
그런데 실제 운영 시 CEO가 줄 수 없는 인풋이 존재한다.

- 레퍼런스 영상/이미지: 핑구만 제공 가능
- 실제 인플루언서에게 결과 전달: 핑구가 직접 실행
- 취향·스타일 기준: 핑구만 앎
- 최종 산출물 실제 적용: 핑구 작업

이것들은 CEO가 "판단 못 해서" 멈추는 게 아니라, 구조적으로 핑구의 행동이 필요한 지점이다.
기존 보고 조건 10개에 이 개념이 없어서 설계 공백이 발생했다.

---

## 결정

핑구의 개입을 두 가지 유형으로 분리한다.

### 유형 1 — 회장 보고 조건 (기존)
CEO가 자율 판단 불가 → 작업 중단 → 핑구 결정 요청

### 유형 2 — 핑구 작업 요청 (신규)
CEO가 제공할 수 없는 인풋·실행이 필요 → 작업 일시 중단 → 핑구가 직접 수행 → 결과 CEO에 전달

---

## 핑구 작업 요청 발생 시점 (MVP 기준)

| 시점 | 요청 내용 | Spiral |
|------|-----------|--------|
| 컨셉 기획 에이전트 실행 전 | 레퍼런스 영상/이미지 (선택, 없으면 생략) | 1 |
| Spiral 0-A 완료 후 | 생성된 프롬프트를 Claude/GPT에 직접 붙여넣어 테스트 | 0-A |
| 최종 리포트 승인 후 | 실제 인플루언서에게 결과 전달 + 적용 | 2 |

---

## 인풋 구조 변경

기존 7개 필드는 유지. 레퍼런스는 별도 선택 필드로 추가.

```python
input = {
    # 기존 7개 필드 (필수)
    "대상자": {
        "이름": str,
        "직업": str,
        "특기": str,
        "성격": str,
        "타겟연령대": str,
        "SNS경험": str,
        "목표": str,
    },
    # 신규 (선택)
    "레퍼런스": {
        "영상": list,   # URL 또는 파일 경로, 없으면 []
        "이미지": list, # URL 또는 파일 경로, 없으면 []
        "메모": str,    # 레퍼런스 설명, 없으면 None
    }
}
```

레퍼런스 없을 시: CEO가 "정보 없음"으로 처리 후 계속 진행. 중단 금지.

---

## 핑구 작업 요청 형식

CEO가 핑구에게 작업을 넘길 때 사용하는 형식.
회장 보고 형식(A/B 선택지)과 구분된다.

```
## 핑구 작업 요청
요청 유형: 작업 요청
요청 내용: {무엇을 해달라는지}
필요한 이유: {왜 CEO가 대신 할 수 없는지}
완료 후 전달 방법: {결과를 어떻게 CEO에게 돌려주는지}
선택 여부: 필수 / 선택 (선택이면 생략 가능)
```

---

## 보고 유형 전체 정리

| 유형 | 트리거 | 핑구에게 필요한 행동 | 해당 조건 |
|------|--------|---------------------|-----------|
| 선택 | CEO가 A/B 결정 못 함 | 선택지 중 고름 | 3, 4, 5, 6 |
| 승인 | 진행 전 확인 필요 | 승인/거부 | 1, 7, 8, 9 |
| 정보제공 | CEO 판단에 인풋 부족 | 추가 정보 제공 | 10 |
| 중단보고 | 계속 진행 불가 | 방향 결정 | 2 |
| **작업요청** | CEO가 실행 불가 | 직접 수행 후 결과 전달 | 신규 |

## 기각된 대안

**대안 1 — 레퍼런스를 필수 8번째 인풋 필드로 추가**
기각 이유: MVP에서 항상 레퍼런스가 있는 게 아님. 없어도 동작해야 함.

**대안 2 — 보고 조건 10개에 핑구 작업 요청을 11번째로 추가**
기각 이유: 개념이 다름. 보고 조건은 "판단 요청", 작업 요청은 "실행 요청". 같은 리스트에 섞으면 혼동.
