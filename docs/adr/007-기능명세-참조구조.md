# ADR 007 — 기능명세 참조 구조 (지도 vs 도면 분리)
date: 2026-05-12
status: 확정
세션: 4

---

## 컨텍스트

기능명세.md에 CEO 설계 + 하위 에이전트 설계 + 컨텍스트 키 표준이 모두 포함되어 있었다.
CEO/ 폴더를 세분화하면서 동일 내용이 두 곳에 존재하는 중복 문제가 발생했다.
구현 시 어떤 문서를 기준으로 삼아야 하는지 불명확해졌다.

---

## 결정

```
기능명세.md     = 지도 (전체 조감도 — 요약 + 참조 링크만)
CEO/            = 상세 도면 (CEO 구현 시 참조, 01~08)
하위에이전트/   = 상세 도면 (각 에이전트 구현 시 참조, 01~04)
```

- 기능명세.md에 내용 직접 작성 금지 — 요약 + 링크만
- 컨텍스트 키 표준 → CEO/05_컨텍스트관리.md로 이동
- 하위 에이전트 섹션 → 하위에이전트/ 폴더로 이동, 기능명세.md는 링크만 유지
- 최종 리포트 파일명 → `최종리포트.md`로 통일 (기존 `report_최종.md` 제거)

---

## 고려한 대안

| 대안 | 제외 이유 |
|------|-----------|
| 기능명세.md에 모든 내용 유지 | 파일 비대화, 수정 시 충돌 발생, CEO/ 폴더와 이중화 |
| CEO/ 폴더만 유지, 기능명세.md 삭제 | 전체 조감도 없으면 새 세션 진입 시 파악 어려움 |

---

## 근거

설계 문서가 길어질수록 전체를 파악하는 지도와 구현에 쓰는 상세 도면을 분리해야
각자의 역할이 명확해진다. 기능명세.md는 새 세션 시작 시 빠르게 전체 구조를 파악하는 용도,
CEO/·하위에이전트/는 실제 시스템 프롬프트 작성 시 참조하는 용도.
