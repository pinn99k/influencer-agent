# ADR 029 — 1장 졸업 선언 + 2장 자율 에이전트 하네스 전환
date: 2026-06-04 (세션 31)
status: accepted

## Context
세션 31에서 핑구-Claude 소크라테스식 대화로 프로젝트의 정체를 재정의했다.
코드 확인 결과: 현재 시스템은 "자율 에이전트"가 아니라 "잘 만든 워크플로우"다.
- 에이전트(BaseAgent)는 도구가 없는 단발 LLM 호출 (base_agent.py).
- CEO.run()은 선형 1패스(_interpret_goal -> _agent_loop -> _finalize), 루프 아님.
- call_llm payload에 tools 없음 (llm_client.py:72) -> 모델이 행동 불가.
즉 "자율 경영 CEO"라는 서사가 실물(고정 파이프라인)을 초과했다.

핵심 통찰: 모델은 생각하나, API 1콜은 무상태 단발 -> 행동/루프/기억 없음.
"데스크톱 클로드가 잘하는 것"은 같은 모델 + 하네스(루프+도구+컨텍스트)일 뿐.
src/는 직접 만든 약한 하네스. 멍청함은 모델이 아니라 하네스 탓.

## Decision
1. 워크플로우 MVP를 "1장"으로 졸업 선언한다 (완성도 90/100).
2. "2장 = 자율 에이전트 하네스"를 새 목표로 명시한다. 맨 API로 도구 호출 루프를
   직접 구현한다. Claude Agent SDK/LangChain 미사용 — 하네스 원리 체득이 목적.
3. 2장 본체 = 에이전트 4요소: 도구 / 루프 / 환경 피드백 / 가드레일.
4. 선결: 트랙 0(CEO 판단 검증)이 필수로 승격. 미검증 판단 위 자율 루프 금지.
5. 향후 계획은 docs/workflow/2장_에이전트하네스/ 폴더에 보관(개요/로드맵/설계스케치).

## Consequences
+ 서사와 실물이 일치 -> 포트폴리오 정직성 확보(검증 규율을 서사 맨 앞으로).
+ 재사용 자산 명확 — core/ 인프라는 하네스 차대, knowledge/는 핵심 자산.
+ 학습 가치 명확 — 하네스를 손으로 지어 에이전트 작동 원리 체득.
- src/의 선형 파이프라인/고정 DAG는 도구 루프로 대체되며 대부분 redundant.
- 최근 작업(관리 UI, 부서)은 부차로 격하 — 우선순위 재배열.
- 비용 증가(루프당 다수 콜) — max_iter/캐시로 관리.

## Alternatives 기각
- 서사를 내려 "워크플로우 기획사"로 종료: 핑구 목표(자율성 체득)와 불일치. 기각.
- Claude Agent SDK로 하네스 대체: 빠르고 배포가능하나 "원리 체득" 학습목표와 반대.
  (단 2장 검증 후 제품화 시 재고 가능 — 졸업이지 회피가 아니므로.)
- 워커별 도구 부여: 1차는 CEO만 도구 보유 권장(복잡도 통제). 워커 도구는 후순위.
