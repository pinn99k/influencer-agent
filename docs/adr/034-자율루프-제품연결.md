# ADR 034 — 2장 자율 도구 호출 루프 제품 연결 (CLI + 웹)
date: 2026-06-16 (세션 38)
status: accepted

## Context
ADR 031에서 도구 호출 루프(CEO.run_autonomous)를 구현·E2E 검증했으나, 제품(CLI·웹)
어디서도 호출되지 않았다. tests/e2e_agent_loop.py만 직접 호출 -> 2장의 심장이 사용자
입장에서 죽어 있었다(ADR 031 Consequences에 "웹/main은 선형 run() -> 자율루프 미연결,
다음 세션 후보"로 명시). 선형 run()과 자율 run_autonomous가 병존하지만 진입점이 없으면
자율 경로는 검증 스크립트 전용이다.

## Decision
자율 루프를 두 진입점에 연결한다. 선형 run()은 기본/폴백으로 유지(병존, 제거 안 함).
1. CLI: main.py에 --autonomous 플래그. 일반 실행에서 (args.autonomous and not dry_run)
   이면 ceo.run_autonomous(context), 아니면 ceo.run(context).
2. 웹 백엔드: JobContext.mode("linear"|"autonomous") + start_job(subject, mode=) +
   _run_ceo가 mode로 분기. job_completed 이벤트에 mode 부착(재분석 mode 패턴과 일관).
3. 라우트: /start(StartRequest.mode), /interview/confirm(ConfirmRequest.mode)이 mode를
   받아 검증(둘 외엔 linear로 안전 폴백) 후 start_job에 전달.
4. 프론트: 인터뷰 확정 바에 '자율 모드' 토글(State.runMode). confirmInterview가 mode를
   interviewConfirm으로 전달. 확정 바는 좁은 채팅 컬럼에서 세로 스택으로 정렬.
5. 테스트 위생: main.py 모듈 레벨 win32 stdout 재래핑이 pytest 캡처를 깨뜨려,
   '"pytest" not in sys.modules' 가드로 테스트 하에서만 건너뜀(실 CLI는 UTF-8 강제 유지).

## Consequences
+ 2장 자율 에이전트가 실제 사용 가능(CLI 플래그 + 웹 토글)해졌다.
+ mode가 라우트->세션->CEO->완료이벤트까지 일관 전달 -> 후속 E단계(워크플로우 vs 에이전트
  A/B)를 실제 시스템 경로로 돌릴 발판.
+ 선형/자율 병존 -> 회귀 위험 최소(기본은 여전히 linear).
- 자율 모드 1회 실행 비용(LLM 콜 다수)은 사용자 선택 시 발생. max_iter 가드레일로 관리.
- 자율 '이득'(워크플로우 대비)은 여전히 미측정 -> E단계 숙제.

## 검증
- 단위: tests/test_autonomous_wiring.py 6개(웹 라우팅 4 + CLI 분기 2). 전체 379 pass(373->379).
- 정적: 4개 JS node --check OK, 변경 파일 한자 0, api.main import OK, --help에 --autonomous 노출.
- 브라우저(preview, 실 LLM 없이 상태 주입): 인터뷰 확정 바 토글 렌더, 클릭 시 State.runMode
  linear<->autonomous + 재렌더 반영, 콘솔 에러 0, 레이아웃 세로 정렬.

## Alternatives 기각
- 자율을 기본값으로: 비용/회귀 위험 -> 기본 linear 유지가 안전. 기각.
- 프론트 토글 생략(API만): 웹 사용자가 자율을 못 고름 -> '웹을 자율루프로'(세션35 옵션2) 미완. 기각.

## 후속 (같은 세션 — 하드닝)
1. F1 수정: run_autonomous가 run()과 달리 성과·방향(_load_direction)을 주입하지 않아
   자율 모드에서 사용자 방향이 누락됐다. run()과 동일하게 _build_ceo_summary 앞에서 주입.
   회귀 가드(tests/test_autonomous_direction.py). 자체 코드리뷰로 발견.
2. T1 관찰성·provenance(로드맵 V단계 성격): 자율 루프의 도구 선택을 감사 가능하게 한다.
   - ToolExecutor가 각 워커 도구 호출을 MeasureStore.log_decision(actor=AI)로 기록
     (focus->basis, 워커->decision). 측정 실패는 파이프라인 비차단(예외 삼킴).
   - AgentLoop가 stop_reason(finish/no_tool/max_iter) 반환 + loop_stopped emit ->
     run_autonomous pipeline_completed에 노출(왜·언제 멈췄나 관찰 가능).
   - tests/test_autonomous_observability.py 6.
3. T0 위생: State.runMode를 새 대상자 reset에서 linear 초기화.
검증: 386 pass(373->386), 한자 0, node --check OK. 실 LLM e2e는 다음 세션.
