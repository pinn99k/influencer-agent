# ADR 033 — GitHub 공개 레포 분리 + 문서 동기화

- 날짜: 2026-06-15 (세션 37)
- 상태: 채택
- 관련: ADR 014(빌드/런타임 레이어 분리), ADR 029(1장 졸업)

## 맥락
- 1장(워크플로우 MVP)이 완료 상태(세션 36, 테스트 373 pass)에 도달.
- 포트폴리오/취준 목적의 공개 GitHub 레포가 필요.
- 현재 프로젝트 폴더의 git이 부모 모노레포(D:\House — 개인 파일 포함)와 얽혀 있어
  in-place push가 위험. 또한 레포에 .env(API 키)·실사용자 산출물(outputs/)·취준 전략
  문서 등 비공개 자산이 섞여 있음.

## 결정
1. 문서 동기화: 공개 문서의 stale 수치를 현재 상태로 일괄 갱신.
   - README.md: 테스트 268->373, ADR 30->32, 측정 레이어 구현완료, 세션 35~36 진행 반영.
   - docs/취준_프로젝트요약.md: 세션 34->36 갱신(코워크 인계용, 비공개 유지).
   - status / MVP범위 / 최종목표: 현황 주석.
2. 새 클린 폴더(influencer-agent)로 복제해 독립 Public 레포로 분리.
   - 제외: .git(부모 얽힘), .env(키), src/outputs(개인 데이터), _e2e_*.txt, 캐시,
     .claude(개인화), 취준_프로젝트요약.md / 벨로그초안(취준 전략), task_08 / Claude_Design(참고용).
   - 포함: src/(코드·테스트·prompts·knowledge, .env.example), docs/(설계·ADR·review·workflow),
     README / LICENSE / CLAUDE.md / .gitignore.
3. README 언어: 한글. 공개 범위: Public.

## 결과
- 모노레포 얽힘·비공개 자산 유출 위험 제거. 복제본 누출 감사 0건 확인.
- 원본 프로젝트는 그대로 유지(복제본만 공개).
