## 결정

Step 3 산출물을 단일 파일이 아닌 두 파일로 분리한다.

- `MVP범위.md` — 현재 구현 범위 기준선 (IN/제약/품질기준/향후확장)
- `최종목표.md` — Phase 8까지 방향 나침반 (구현 명세 아님)

## 이유

- MVP와 최종 목표를 같은 파일에 두면 구현 중 범위 혼동 발생 가능
- "지금 뭘 만드는가"와 "어디까지 갈 건가"는 독자와 사용 시점이 다름
  - MVP범위.md: 구현 중 흔들릴 때 참조 → 기준선 고정
  - 최종목표.md: Phase 계획·확장 방향 검토 시 참조 → 나침반

## 기각된 옵션

- 단일 파일에 섹션 분리: 파일이 길어질수록 기준선 역할 약화
- task_08 기존 파일 참조 유지: 워크플로우 폴더 외부 참조는 세션 로드 누락 위험
