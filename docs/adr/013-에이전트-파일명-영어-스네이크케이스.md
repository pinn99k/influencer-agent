# ADR 013 — 에이전트 파일명·클래스명 영어 snake_case

date: 2026-05-13
status: 확정

---

## 문제

기능명세(step4)에서 에이전트 파일명을 `01_대상분석.py` 형식으로 정의했으나
Python은 숫자로 시작하는 모듈명 import를 허용하지 않음.

```python
from agents.01_대상분석 import ...  # SyntaxError
```

## 결정

에이전트 파일명과 클래스명을 **영어 snake_case / PascalCase**로 변경.
한글 표시명은 클래스 속성 `display_name`으로 보존.

| 기존 (명세) | 확정 (구현) | 클래스명 | display_name |
|------------|-----------|---------|-------------|
| `01_대상분석.py` | `subject_analyst.py` | `SubjectAnalystAgent` | `"대상분석"` |
| `02_경쟁분석.py` | `competition_analyst.py` | `CompetitionAnalystAgent` | `"경쟁분석"` |
| `03_플랫폼추천.py` | `platform_recommender.py` | `PlatformRecommenderAgent` | `"플랫폼추천"` |
| `04_컨셉기획.py` | `concept_planner.py` | `ConceptPlannerAgent` | `"컨셉기획"` |

프롬프트 템플릿 파일(.md)도 동일하게 영어 파일명 사용.

| 기존 | 확정 |
|------|------|
| `01_대상분석.md` | `subject_analyst.md` |
| `02_경쟁분석.md` | `competition_analyst.md` |
| `03_플랫폼추천.md` | `platform_recommender.md` |
| `04_컨셉기획.md` | `concept_planner.md` |
| `ceo_목표해석.md` | `ceo_goal_interpretation.md` |
| `ceo_다음판단.md` | `ceo_next_decision.md` |

## 근거

- Python 표준: 모듈명은 소문자 + 숫자 가능하나 숫자 시작 불가 (PEP 8)
- 한글 클래스명은 기술적으로 가능하나 IDE·linter·일부 도구에서 문제 발생 가능
- `display_name` 속성으로 한글 표시명 보존 → state.md 출력, AGENT_KEY_MAP에 사용
