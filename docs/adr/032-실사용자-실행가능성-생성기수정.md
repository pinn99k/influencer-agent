# ADR 032 — 실사용자 실행가능성 기준 + 생성기 결함 수정
date: 2026-06-14 (세션 36)
status: 채택

## 배경
실사용자(예으니)에게 산출물을 보였더니 "뭘 하라는지 모르겠다, 못 쓰겠다"는 피드백.
370 테스트 + 웹 E2E 통과 상태였는데도 실사용 불가. 통과 신호가 품질을 보장하지 못했다.

## 문제 (예으니 산출물 실제 결함)
1. 최종리포트 전체가 "예은이"(타인/오타) — 대상자 이름 오염.
2. 컨셉 산출물에 플레이스홀더 "정보 없음" 노출.
3. 성공지표 팔로워 1,000/조회수 5,000 날조. 본인 목표(1만)와도, 현실 기준(300)과도 불일치.
4. 콘텐츠 목록이 3개(영상아이디어/4주캘린더/리포트로드맵) 제목이 제각각 -> "뭘 할지" 혼란.

## 근본 원인
- "PASS"의 정의가 OutputValidator(섹션/키워드/길이 = 구조)였다. 실행가능성/피드백 충족이 아님.
- _judge_quality는 출력 앞 500자만 보고 채점 -> 느슨한 "5/5 PASS".
- 최종리포트 생성에 이름/KPI 검증 게이트 부재.
- 컨셉 프롬프트가 아이디어와 캘린더를 별개 목록으로 생성, 리포트가 또 새 로드맵 생성.

## 결정
1. 채점 기준을 "실행가능성"으로 재정의 — docs/실전적용/실행가능성_루브릭.md (5영역).
   OutputValidator는 전제조건일 뿐, 사람이 루브릭으로 별도 채점.
2. 생성기 소스 수정 (전 대상자 적용):
   - concept_planning.md: 플레이스홀더 가드, 아이디어<->캘린더 단일 계획, 해시태그 예시복사 금지,
     촬영 실제환경 기반, 추천형/어그로 금지 강화.
   - final_report.md: 이름 정확/축약 금지, KPI 현실 앵커(300/500)+장기목표 분리, 로드맵=캘린더 요약.
   - ceo.py: 최종리포트 이름 게이트+1회 재시도, payload에 방향 주입, 판정기 미리보기 500->3000.
   - output_validator.py: 컨셉기획 "정보 없음" 누출 결정론적 가드 + 회귀 테스트 3개.

## 검증
실 gpt-4o로 예으니 재생성(--reanalyze) -> 4종 결함 해소(이름 일관, 플레이스홀더 0,
KPI 300/12/조회수[추정]/장기목표1만, 아이디어->캘린더->로드맵 일치). 373 pass.

## 결과
- 긍정: 실사용 차단 결함 제거. 소스 수정이라 향후 모든 대상자에 적용.
- 한계: 깊이(촬영 개인화, 캡션/해시태그)는 보통. LLM이 일반론을 일부 유지. 폴리시는 후속.
- 교훈: 그린 테스트로 "보여줄 만하다"고 단정 금지. 실사용자 실제 산출물로 채점한다.

## 참조
- 루브릭: docs/실전적용/실행가능성_루브릭.md
- 예으니 피드백: src/outputs/예으니/피드백.md


## 후속 (세션 36) — 사용자 여정 평가 + 운영 레이어
실행가능성을 점수화. tests/e2e_user_journey.py: LLM이 대상자 본인을 연기하며 10단계를
0/1/2 채점 + 결정론적 체크. PASS 기준 = 총점 >= 16/20 AND 막힌 단계(0점) 없음 AND 결정론 통과.

- 1차 예으니 13/20 FAIL: 콘텐츠 제작(1~7)은 가능하나 피드백 반영/차주 조정(9,10)=0.
  원인 = "한 달 운영/조정 루프"가 사용자 산출물에 없음(운영키트는 운영자용).
- 결정: 산출물에 사용자용 운영 레이어 추가.
  * final_report.md: ## 매주 반복 루틴 / ## 성과 기록 / ## 2주 후 셀프 점검 3섹션.
  * ceo.py 최종리포트 필수 게이트에 3섹션 추가.
  * concept_planning.md: 영상 구성(훅/핵심/마무리), 해시태그 최소개수, 편집 3단계.
  * 4개 워커 프롬프트 영어 번역 제약(업스트림 영어 발원 차단).
  * 최종리포트 생성모델 DEFAULT(groq) -> gpt-4o (가장 중요한 문서 품질).
- 2차 예으니 16/20 PASS: 막힌 단계 0, 결정론 전부 통과. 잔여(비차단) = 편집 깊이/
  캘린더 변주 개념/분석 숫자 해석.
- 교훈 보강: 구조 채점뿐 아니라 "사용자 역할 연기 채점"을 회귀로 둔다(e2e_user_journey).
