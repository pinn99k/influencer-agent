# ADR 031 — 2장 Phase A(디렉티브 seam) + Phase C(도구 호출 루프)
date: 2026-06-14 (세션 35)
status: accepted

## Context
설계 재감사에서 5갭 발견. 핵심: G1(CEO plan의 per-agent 지시가 plan.md에만 남고 워커에
미도달 — 고아), G4(ceo_summary가 전략의 단일 LLM 문자열 깔때기 — 취약, 깨지면 4워커 오염),
G2(워커 멀티스텝 죽은코드). 또 ADR 029가 짚은 "call_llm payload에 tools 없음 -> 모델 행동
불가"가 2장(자율 에이전트)의 본질적 공백.

## Decision
1. Phase A — AgentDirective: 구조화 per-agent 지시 패킷을 도입한다.
   - _interpret_goal이 plan 표를 파싱해 per-agent 지시를 런타임 context['에이전트_지시']로 복원.
   - _build_ceo_summary가 context['directives'](agent->directive text) 생성.
   - build_context가 각 워커에 자기 directive를 ceo_summary 키로 주입(워커/프롬프트 무변경).
   - 방향은 DirectionProfile 구조화 주입(올-미정 차단). 이 디렉티브가 Phase 2 도구 호출 payload로
     그대로 승격된다.
2. Phase C — 도구 호출 루프: 맨 API로 직접 구현(Agent SDK 미사용, ADR029 일관).
   - call_llm_tools 프리미티브(OpenAI tool-calling) 추가.
   - core/tools.py: WORKER_TOOLS + ToolExecutor(워커를 run_partial로 재사용 -> 검증·저장 그대로).
   - core/agent_loop.py: AgentLoop(루프 + max_iter 가드레일 + finish/무도구 정지).
   - CEO.run_autonomous로 구동. 선형 run()은 폴백 드라이버로 유지(병존, 제거 안 함).
3. 컨셉 출력 FORMAT 심화: 프롬프트 OUTPUT 구조가 품질 천장이므로 구조를 연다(해시태그 3층+이유,
   영상 편집 컨셉 섹션, 캡션 복수, 날조·소재지시 금지). 실 사용자 피드백으로 검증.

## Consequences
+ 시스템이 처음으로 도구+루프+피드백+가드레일을 갖춘 '에이전트'로 작동(통합 실LLM 확인).
+ 워커 재작성 없이 seam만으로 Phase 2 진입 — AgentDirective가 파이프라인/루프 공용 payload.
+ 고아 지시 해소로 CEO 전략이 산출물에 온전히 흐름. 출력 깊이/피드백 반영 개선(예으니 5/5).
- 자율성 '작동'은 검증, '이득'(워크플로우 대비)은 미측정 -> 로드맵 E단계 A/B 비교 필요.
- 루프 비용 증가(콜 다수) -> max_iter 상한으로 관리.
- 웹/main은 아직 선형 run() -> 자율루프 미연결(다음 세션 후보).

## Alternatives 기각
- Agent SDK로 하네스 대체: 빠르나 '원리 체득' 학습목표와 반대(ADR 029 일관). 기각.
- ceo_summary 단일 문자열 유지: 취약+고아 미해결. 기각.
- 컨셉 멀티스텝 즉시 재활성화: 깊이엔 유리하나 비용/검증 부담 -> 프롬프트 FORMAT 심화를
  먼저(저위험·검증가능). 멀티스텝은 Phase B로 보류.
