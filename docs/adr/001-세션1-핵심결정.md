# ADR 001 — 세션1 핵심 결정사항
date: 2026-05-12
status: 확정
step: 0~2 (문제정의·성공기준·사용자플로우)

## 결정

세션 1에서 확정된 5개 결정사항.

**1. 무상태 시작 방식**
핑구는 목표·목적 + 대상자 기본 양식만 제시. 나머지는 경영 에이전트가 자율 판단·관리.

**2. 정보 부족 처리 — B방식**
하위 에이전트가 정보 부족 감지 시 → 경영 에이전트에 플래그 → 핑구에게 질문 → 답변 후 재개.

**3. 컨텍스트 전달 — Structured 방식**
경영 에이전트가 각 에이전트 결과를 섹션별로 정리 → 다음 에이전트에 필요한 섹션만 전달.

**4. MVP 품질 기준**
MVP부터 실제 사람에게 적용. 구조 검증 + 실사용 품질 동시 충족. 둘 중 하나만이면 MVP 실패.

**5. 워크플로우 폴더 구조**
docs/workflow/step{N}_{이름}/ 폴더 구조. 참고 파일은 docs/refs/.

## 이유

- 에이전트가 스스로 판단·관리해야 확장 가능. 사람 개입 최소화.
- 정보 부족 상태로 진행하면 실사용 품질 타협 발생
- Structured 방식: Full 대비 토큰 80~90% 절감, Summarized 대비 정보 손실 없음
- step별 파일이 많아질 때 flat 구조는 관리 불가

## 기각된 옵션

- A방식 (미정으로 표시 후 진행): 품질 기준 미충족
- Full 컨텍스트 전달: 토큰 낭비
- Summarized 컨텍스트: 정보 손실
- flat 파일 구조: 에이전트가 필요한 것만 참조 불가
