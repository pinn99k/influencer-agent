# 운영관리 — CEO 판단 프레임워크
updated: 2026-05-16

## 사고흐름: 실행 가능성 평가

CEO가 전략을 세운 후, "이 전략이 실제로 실행 가능한가"를 평가하는 흐름.

### Step 1. 가용 리소스 파악
```
질문: 이 사람이 콘텐츠 제작에 투입할 수 있는 현실적 리소스는?
판단 근거:
  - 본업이 있는가? → 있으면 주당 가용 시간 제한적
  - 장비/스킬이 있는가? → 없으면 초기 학습 기간 필요
  - 비용 투자 가능한가? → 제한적이면 무료 도구 중심 전략
  - 정보 없으면 → "정보 부족" 명시, 보수적 가정
출력: 가용 리소스 요약 (시간/장비/스킬/비용)
```

### Step 2. 콘텐츠 사이클 추정
```
질문: 제안하려는 콘텐츠 유형의 1회 생산에 얼마나 걸리는가?
판단 방법:
  - 콘텐츠 유형별 표준 사이클 참조
  - 초보자면 표준의 1.5~2배로 보정
  - 경험자면 표준 그대로 적용
출력: 예상 사이클 시간
```

### Step 3. 적정 빈도 산출
```
질문: 번아웃 없이 유지 가능한 업로드 빈도는?
판단 방법:
  - 적정 빈도 = 가용 시간 / 사이클 시간 x 0.7
  - 주 1회 미만이면 → 더 가벼운 콘텐츠 유형 고려
  - 주 3회 이상이면 → 숏폼 아니면 비현실적
출력: 주간 업로드 빈도 권장값
```

### Step 4. 전략-실행 정합성 체크
```
질문: 전략이 현실적 제약 안에서 실행 가능한가?
판단 방법:
  - 전략이 요구하는 빈도 > 적정 빈도 → 전략 수정 필요
  - 전략이 요구하는 스킬 > 현재 스킬 → 학습 기간 고려
  - 전략이 요구하는 비용 > 가용 예산 → 대안 모색
  - 모두 적합하면 → 전략 확정
출력: "실행 가능" / "조정 필요 (구체적 항목)"
```

## 판단 원칙
- 이상적 전략보다 실행 가능한 전략이 우선
- 정보 부족 시 보수적으로 가정 (낙관적 추정 금지)
- 리소스 제약은 에이전트에게 반드시 전달 — 비현실적 제안 방지
- 번아웃은 크리에이터 최대 리스크 — 빈도 설정에서 항상 여유 확보
