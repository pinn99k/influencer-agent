# 커뮤니케이션 — 핵심 도메인 지식
updated: 2026-05-16

## 1. 보고 구조

### 피라미드 원칙 (결론 먼저)
```
결론 → 근거 → 세부 사항
```

CEO가 회장에게 보고할 때, 에이전트에게 지시할 때 모두 이 구조를 따른다.

### 보고문 필수 요소
| 요소 | 설명 | 예시 |
|------|------|------|
| 보고 조건 | 왜 보고하는가 (10가지 중) | "조건 6: 에이전트 간 결과 충돌" |
| 현재 상황 | 무슨 일이 발생했는가 (팩트) | "대상분석은 '내성적'이나 컨셉은 '활발한 브이로그' 추천" |
| 영향 | 이대로 진행하면 어떻게 되는가 | "크리에이터에게 맞지 않는 전략 전달 위험" |
| 선택지 | 회장이 고를 수 있는 옵션 | "A: 대상분석 재실행 / B: 컨셉 방향 수정" |
| 추천 | CEO의 의견 (선택지 중 하나) | "A 추천 — 근거: 대상분석 정보가 더 신뢰" |

### 나쁜 보고 vs. 좋은 보고
| 나쁜 보고 | 좋은 보고 |
|----------|----------|
| "문제가 생겼습니다" | "조건 5 해당: 경쟁분석 2회 실패" |
| "어떻게 할까요?" | "A: 재시도(비용 추가), B: 건너뛰기(품질 저하). A 추천" |
| 긴 배경 설명 후 결론 | 결론 먼저, 배경은 필요 시 |

## 2. 선택지 제시법

### 선택지 설계 원칙
- 최소 2개, 최대 3개 (선택 과부하 방지)
- 각 선택지에 트레이드오프 명시 (장점 + 단점)
- CEO 추천 포함 (근거와 함께)
- "아무것도 안 한다"도 유효한 선택지

### 선택지 프레임
```
A. [행동] — 장점: ..., 단점: ...
B. [행동] — 장점: ..., 단점: ...
CEO 추천: [A/B] — 이유: ...
```

## 3. 지시 커뮤니케이션

### 에이전트에게 rework 지시할 때
| 나쁜 지시 | 좋은 지시 |
|----------|----------|
| "더 잘 해" | "강점 분석에 대상자 직업 경험을 근거로 추가" |
| "구체적으로 해" | "경쟁자 3명의 실명과 구독자 수를 포함" |
| "다시 해" | "섹션 2의 약점 분석이 generic — 대상자의 활동 기간(6개월)을 반영해 재작성" |

구체적이지 않은 지시는 같은 실패를 반복시킨다.
