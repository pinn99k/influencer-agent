# 재무관리 — CEO 판단 프레임워크
updated: 2026-05-16

## 사고흐름: 비용/ROI 판단

CEO가 비용이 발생하거나 리소스 배분이 필요한 결정을 할 때의 사고흐름.

### Step 1. 비용 발생 여부 확인
```
질문: 이 결정으로 추가 비용이 발생하는가?
체크 대상:
  - API 호출 횟수 증가 (Groq rate limit, Serper 월 한도)
  - 모델 변경 (무료 → 유료)
  - 외부 서비스 추가
  - 시간 비용 (핑구 또는 크리에이터의 시간)
결과:
  - 비용 없음 → 재무 판단 불필요, 다른 기준으로 결정
  - 비용 있음 → Step 2로
```

### Step 2. 비용 규모 산정
```
질문: 얼마가 드는가? (구체적 수치)
판단 방법:
  - API: 호출 횟수 x 단가
  - 시간: 소요 시간 x 시급 환산
  - 서비스: 월 구독료
  - 불명확하면 → 최대값으로 추정 (보수적)
출력: 예상 비용 (원/달러)
```

### Step 3. 기대 효과 추정
```
질문: 이 비용으로 무엇을 얻는가?
판단 방법:
  - 품질 향상: generic → specific 전환 가능?
  - 시간 절약: 수동 작업 대비 몇 시간 절감?
  - 범위 확대: 더 많은 크리에이터 서비스 가능?
출력: 기대 효과 1~2줄
```

### Step 4. ROI 판정
```
질문: 비용 대비 효과가 충분한가?
판단 로직:
  - 비용 0 + 효과 있음 → 무조건 진행
  - 비용 소액 + 효과 명확 → CEO 판단으로 진행
  - 비용 중액 이상 → 회장 보고 (조건 2)
  - 효과 불명확 → 소액이어도 보류, 무료 대안 탐색
출력: "진행" / "보류" / "회장 보고"
```

## 판단 원칙
- MVP 단계에서 유료 전환은 극도로 신중하게 — 무료로 안 되는 증거 먼저
- "더 좋아질 것 같다"는 투자 근거가 아니다 — 구체적 효과 추정 필수
- 비용 발생은 규모와 무관하게 기록 — 누적 비용 추적
- 시간 비용도 비용이다 — 핑구의 시간은 가장 비싼 리소스
