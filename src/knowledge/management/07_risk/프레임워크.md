# 리스크관리 — CEO 판단 프레임워크
updated: 2026-05-16

## 사고흐름: 리스크 감지 및 대응

CEO가 에이전트 결과물이나 상황 변화에서 리스크를 감지하고 대응하는 흐름.

### Step 1. 리스크 스캔
```
질문: 현재 상황에서 감지되는 리스크가 있는가?
스캔 대상:
  - 에이전트 산출물에 법적 문제 소지 (저작권, 초상권, 광고법)
  - 전략이 대상자의 현실적 제약을 초과
  - 에이전트 간 결과가 모순 (예: 대상분석은 "내성적"인데 컨셉은 "활발한 브이로그")
  - 비용/리소스가 예산을 초과
출력: 감지된 리스크 목록 (없으면 "리스크 없음")
```

### Step 2. 리스크 분류
```
질문: 감지된 리스크의 유형과 심각도는?
판단 방법:
  - 리스크 매트릭스 참조하여 유형 분류
  - 심각도 판단: 최고/높음/중간/낮음
출력: 각 리스크별 유형 + 심각도
```

### Step 3. 대응 결정
```
질문: CEO가 자체 대응할 것인가, 회장에게 보고할 것인가?
판단 로직:
  - 심각도 "최고" → 즉시 작업 중단 + 회장 보고 (조건 1번)
  - 심각도 "높음" → 회장 보고 (해당 조건 번호 매핑)
  - 심각도 "중간" → CEO 자체 대응 (에이전트 rework 지시)
  - 심각도 "낮음" → 메모만 남기고 진행
출력: "자체 대응" + 구체적 조치 / "회장 보고" + 조건 번호
```

### Step 4. 보고 조건 매칭 (회장 보고 시)
```
질문: 10가지 보고 조건 중 어디에 해당하는가?
판단 방법:
  - 법적 → 조건 1
  - 비용 → 조건 2
  - 전략 전환 → 조건 3
  - 정보 핵심 누락 → 조건 4
  - 반복 실패 → 조건 5 (자동)
  - 결과 충돌 → 조건 6
  - 외부 변화 → 조건 7
  - 평판 → 조건 8
  - 최종 승인 → 조건 9 (자동)
  - 기대 괴리 → 조건 10
출력: 조건 번호 + 보고 내용 구성
```

## 판단 원칙
- 리스크 판단은 보수적으로 — 애매하면 높은 심각도로 분류
- 법적 리스크는 예외 없이 즉시 중단 + 회장 보고
- "괜찮을 것 같다"는 리스크 관리가 아니다 — 근거 없는 낙관 금지
- 리스크 0은 없다 — "리스크 없음"이더라도 스캔은 매번 수행
