# 의사결정 — 핵심 도메인 지식
updated: 2026-05-16

## 1. 의사결정 바이어스

CEO가 빠지기 쉬운 판단 오류 목록.

### 주요 바이어스
| 바이어스 | 설명 | 인플루언서 도메인 예시 |
|---------|------|---------------------|
| 확증 편향 | 가설에 맞는 증거만 보기 | "이 사람은 유튜브가 맞아"라고 정한 후 불리한 데이터 무시 |
| 매몰 비용 | 이미 투자한 것에 집착 | 에이전트가 2번 분석했으니 결과를 억지로 사용 |
| 가용성 편향 | 쉽게 떠오르는 사례에 의존 | 최근 성공 사례를 모든 대상자에게 적용 |
| 앵커링 | 첫 정보에 과도하게 영향 | 대상자 첫 인상(직업)에 전략 고정 |
| 과신 편향 | 불확실한 판단에 과도한 확신 | 정보 부족한데 확정적 전략 제시 |

### 바이어스 방지 원칙
- 가설과 반대되는 증거를 의식적으로 탐색
- "이 판단의 근거가 무엇인가?" 항상 자문
- 정보 부족 시 [추정] 태그 필수 — 확신하지 않기

## 2. 정보 부족 시 판단법

### 정보 충분성 3단계
| 단계 | 설명 | CEO 행동 |
|------|------|---------|
| 충분 | 판단에 필요한 핵심 정보 존재 | 결정 진행 |
| 부분적 | 일부 정보 부족하나 합리적 추정 가능 | [추정] 태그 + 결정 진행 |
| 불충분 | 핵심 정보 자체가 없음 | 결정 보류 → 정보 수집 지시 or 회장 보고 |

### 핵심 정보 vs. 보조 정보
- **핵심**: 이 정보 없이는 전략 방향이 달라지는 것 (직업, 목표, 현재 활동)
- **보조**: 없어도 방향은 같고 디테일만 달라지는 것 (취미, 성격 세부)

핵심 정보 부족 시 추측 금지. 보조 정보 부족 시 합리적 추정 허용 ([추정] 태그).

## 3. 확신도 기준

### CEO 확신도 스케일
| 확신도 | 의미 | 표현 |
|--------|------|------|
| 높음 | 데이터와 논리가 일치 | 단정적 서술 |
| 중간 | 합리적이나 검증 필요 | [추정] 태그 |
| 낮음 | 근거 불충분 | "정보 부족" 명시 |
| 없음 | 판단 불가 | 결정 보류, 회장 보고 |

확신도 "중간" 이하가 핵심 판단에 3개 이상이면 → 전체 전략 신뢰도 의심 → 회장 보고 고려.
