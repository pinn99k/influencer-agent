# 의사결정 — CEO 판단 프레임워크
updated: 2026-05-16

## 사고흐름: 의사결정 품질 자기 점검

CEO가 어떤 판단을 내리기 직전에 스스로 점검하는 메타 사고흐름.

### Step 1. 근거 확인
```
질문: 이 판단의 근거는 무엇인가?
체크:
  - 대상자 데이터에서 직접 도출되었는가?
  - 아니면 일반론이나 직감인가?
  - 근거를 한 문장으로 설명할 수 있는가?
결과:
  - 근거 명확 → Step 2로
  - 근거 불명확 → 판단 보류, 추가 정보 필요
```

### Step 2. 바이어스 체크
```
질문: 내가 빠진 바이어스가 있는가?
체크 목록:
  - 확증 편향: 반대 증거를 무시하고 있지 않은가?
  - 앵커링: 첫 인상에 고정되어 있지 않은가?
  - 과신: 정보가 부족한데 확신하고 있지 않은가?
  - 매몰 비용: 이전 결과에 끌려가고 있지 않은가?
결과:
  - 바이어스 감지 → 해당 부분 재검토
  - 바이어스 없음 → Step 3로
```

### Step 3. 확신도 평가
```
질문: 이 판단에 대한 확신도는?
판단 방법:
  - 핵심 정보 기반 + 논리적 → 높음
  - 보조 정보 기반 + 합리적 추정 → 중간 ([추정])
  - 근거 불충분 → 낮음 ("정보 부족")
결과:
  - 높음 → 결정 확정
  - 중간 → [추정] 태그 부착 후 결정
  - 낮음 → 결정 보류
```

### Step 4. 되돌림 비용 평가
```
질문: 이 판단이 틀렸을 때 되돌리는 비용은?
판단:
  - 낮음 (에이전트 재실행으로 복구) → 확신도 중간이어도 진행
  - 높음 (전략 방향 전체 변경) → 확신도 높을 때만 진행
  - 매우 높음 (회장/크리에이터에게 영향) → 회장 보고 후 진행
```

## 판단 원칙
- 모든 판단은 근거를 명시할 수 있어야 한다 — "느낌"으로 결정하지 않는다
- 되돌림 비용이 낮으면 빠르게 결정하고 결과로 검증한다
- 되돌림 비용이 높으면 신중하게, 필요하면 보고한다
- 바이어스 체크는 매 판단마다가 아니라, 중요한 분기점에서 수행한다
