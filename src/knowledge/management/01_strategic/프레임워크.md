# 전략경영 — CEO 판단 프레임워크
updated: 2026-05-16

## 사고흐름: 초기 전략 수립

대상자 정보를 받았을 때 CEO는 아래 순서로 사고한다.

### Step 1. 현재 위치 파악
```
질문: 이 사람은 크리에이터 성장 단계 중 어디인가?
판단 근거: 현재 구독자 수, 활동 기간, 콘텐츠 양
출력: 단계 라벨 (진입기/성장기/확장기/안정기)
```

### Step 2. 포지셔닝 진단
```
질문: 세 축(전문성/캐릭터/타겟)이 명확한가?
판단 방법:
  - 전문성: 직업/경험에서 전문 분야가 도출되는가?
  - 캐릭터: 성격/가치관에서 고유 캐릭터가 보이는가?
  - 타겟: 누구에게 가치를 줄 수 있는가?
출력: 각 축별 명확/모호/정보부족 판정
```

### Step 3. 차별화 가능성 평가
```
질문: 어떤 차별화 유형이 가장 자연스러운가?
판단 방법:
  - 이미 고유한 교차점이 있으면 → 크로스오버
  - 전문성이 깊으면 → 니치 전문화
  - 전문성은 보통이나 캐릭터가 강하면 → 캐릭터 차별화
  - 위 모두 약하면 → 포맷 혁신으로 돌파구 탐색
출력: 1순위 차별화 유형 + 근거
```

### Step 4. 전략 방향 결정
```
질문: 이 단계에서 이 사람에게 가장 중요한 한 가지는?
판단 방법:
  - 진입기 → "누구에게 무엇을 말할 것인가" 정의
  - 성장기 → "어떻게 일관되게 만들 것인가" 시스템
  - 확장기 → "어떻게 수익과 규모를 키울 것인가" 구조
  - 안정기 → "어떻게 지속 가능하게 할 것인가" 체계
출력: 전략 방향 한 문장 + 하위 에이전트에게 줄 핵심 지시
```

### Step 5. 가설 수립
```
질문: 이 전략이 맞다면, 어떤 결과가 나와야 하는가?
출력:
  - 강점 가설 1~2개 (에이전트가 검증할 것)
  - 약점 가설 1~2개 (에이전트가 보완할 것)
  - 검증 불가능한 항목은 [추정] 태그
```

## 판단 원칙
- 정보가 부족하면 추측하지 말고 "정보 부족"으로 명시 → 에이전트에게 탐색 지시
- 단계 착오 방지: 현재 단계에 맞지 않는 전략은 제안하지 않는다
- 한 번에 모든 것을 해결하려 하지 않는다 — MVP는 "첫 방향 설정"이 목표
