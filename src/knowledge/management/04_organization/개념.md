# 조직인사관리 — 핵심 도메인 지식
updated: 2026-05-16

## 1. 에이전트 역할 분담

### 현재 조직 구조
```
회장 (핑구) — 최종 의사결정, 방향 제시
  └── CEO — 오케스트레이터, 판단, 위임
        ├── 대상분석 에이전트 — 크리에이터 강점/약점/차별점
        ├── 경쟁분석 에이전트 — 경쟁자 분석, 차별화 포인트
        ├── 플랫폼추천 에이전트 — 최적 플랫폼 + 전략
        └── 컨셉기획 에이전트 — 콘텐츠 컨셉 + 실행 계획
```

### 역할 경계 원칙
| 원칙 | 설명 |
|------|------|
| 단일 책임 | 한 에이전트 = 한 역할. CEO가 분석을 직접 하지 않는다 |
| 역할 침범 금지 | 대상분석이 플랫폼을 추천하면 역할 침범 |
| CEO 독점 영역 | 에이전트 순서 결정, 품질 판단, 회장 보고 |
| 에이전트 독점 영역 | 실제 분석/기획 산출물 생성 |

### 역할 침범 감지 기준
- 에이전트 출력에 다른 에이전트 영역의 결론이 포함됨
- CEO가 에이전트 결과를 직접 수정하여 전달
- 에이전트가 CEO 판단(전략 방향, 회장 보고)을 스스로 수행

## 2. 위임 원칙

### 효과적 위임의 3요소
| 요소 | 설명 | CEO 적용 |
|------|------|---------|
| 명확한 목표 | 무엇을 달성해야 하는가 | 에이전트별 핵심 지시 |
| 권한 범위 | 어디까지 스스로 결정하는가 | 시스템 프롬프트 CONSTRAINTS |
| 결과 기준 | 어떻게 되면 완료인가 | OutputValidator RULES |

### 나쁜 위임 패턴
| 패턴 | 문제 | 개선 |
|------|------|------|
| 과소 위임 | CEO가 세부 지시를 너무 많이 | 목표만 주고 방법은 에이전트 자율 |
| 과다 위임 | "알아서 해"식 지시 | 목표 + 핵심 제약 + 품질 기준 명시 |
| 결과 무시 | 산출물 검토 없이 다음으로 | 매 산출물 품질 판단 필수 |

## 3. 성과 평가 기준

### 에이전트 성과 지표
| 지표 | 측정 | 기준 |
|------|------|------|
| 1회 통과율 | rework 없이 합격한 비율 | 목표: 70% 이상 |
| 품질 일관성 | 대상자별 품질 편차 | 편차 작을수록 좋음 |
| 실행 가능성 | 크리에이터가 실제 사용 가능한 비율 | 목표: 90% 이상 |

### 성과 부진 대응
| 통과율 | CEO 행동 |
|--------|---------|
| 70% 이상 | 정상 운영 |
| 50~70% | 프롬프트 튜닝 검토 |
| 50% 미만 | 프롬프트 전면 재설계 or 모델 교체 검토 → 회장 보고 |
